## 1. 设计目的

锈犬是玩家在 P0 阶段遇到的第一个敌人，承担“弹反教学”和“近战追击敌人基准线”两项职责。

它要让玩家在低压力下理解 Reforge 的基础战斗语法：

```text
识别敌人前摇 → 选择弹反或闪避 → 成功应对 → 获得输出窗口或能量收益
```

第一个敌人需要提供清楚前摇、稳定节奏和宽松弹反窗口，让玩家在低压力下建立第一次成功弹反经验。P0 的主要回能来源是弹反，锈犬的首要职责是帮助玩家理解能量循环。

锈犬同时承担近战追击压力。它会主动发现并追击玩家，进入攻击范围后执行带前摇的扑击。玩家如果不处理它，会被持续逼位；玩家如果学会读前摇，就能把它的攻击转化为弹反收益。这个关系是后续腐鹿、巡墙甲、异铸骑士等节奏型敌人的基础。

---

## 2. 系统定位

锈犬定位为：

| 定位 | 说明 |
|------|------|
| 区域 | 荒野区基础敌人 |
| 类型 | 节奏型 / 近战追击 / 弹反教学 |
| P0 职责 | 验证敌人发现、追击、攻击、伤害、被弹反、受伤、死亡、清房事件 |
| 玩家主要应对 | 弹反、闪避、普攻输出 |
| 后续生态位 | 作为近战追击敌人的基准线 |

和其他敌人的区分：

| 敌人 | 相对锈犬的定位 |
|------|----------------|
| 腐鹿 | 更慢、更重、更痛，教学重击和撞墙破绽 |
| 巡墙甲 | 更硬、更稳定，是中期标准弹反怪 |
| 甲鼠 | 更弱、更快、数量更多，逼迫闪避和清群 |
| 残弩 | 远程蓄力压力，弹反窗口更短 |

P0 阶段只需要单只或少量锈犬。成群交错扑击属于 P1 以后用于制造组合压力的扩展。

## 3. 具体规则

所有时间数值以秒为准。实现和验收不以画面刷新次数为判定单位。

### 3.1 基础行为循环

[巡逻 / 待机]

触发条件：玩家不在检测范围内。

序列：
1. 锈犬在出生点附近待机或沿短路径巡逻。
2. 周期性检测玩家距离。
3. 玩家进入检测范围后，转入追击。

结束后：进入 Chase。

设计理由：P0 可先用待机代替复杂巡逻，但锈犬必须表现出“原本是巡逻军犬”的行为来源。后续做房间布局时，可把巡逻点作为增强项。

[追击]

触发条件：玩家进入检测范围，且锈犬未处于攻击、硬直或死亡状态。

序列：
1. 锈犬朝玩家移动。
2. 持续更新朝向。
3. 玩家进入攻击范围，且攻击冷却结束时，进入后蹲蓄力。
4. 玩家离开检测范围时，返回巡逻 / 待机。

结束后：进入 Charge 或 Patrol。

设计理由：追击给玩家移动压力，但速度不应过高。锈犬的核心威胁来自清晰扑击与稳定前摇。

[铁甲扑击]

触发条件：玩家在攻击范围内，且攻击冷却结束。

序列：
1. 后蹲蓄力（约 0.45 秒）→ 停止移动或明显减速，播放后蹲 / 低吼 / 身体压低表现，锁定扑击方向。
2. 扑击判定（约 0.27 秒）→ 朝锁定方向扑出，敌方 Hitbox 激活，标记为可弹反，命中玩家造成 8 点伤害。
3. 落地恢复（约 0.20 秒）→ Hitbox 关闭，播放落地 / 收势表现。
4. 攻击间隔补足 → 从本次攻击开始到下一次攻击开始至少间隔约 2.0 秒。

结束后：若玩家仍在检测范围内，回到 Chase；否则回到 Patrol。

设计理由：0.45 秒前摇足够让新手看见“它要扑了”；0.27 秒可弹反窗口宽松，用于建立成功经验。攻击间隔稳定，玩家能在 1-2 次交手内读懂节奏。

[被弹反]

触发条件：锈犬处于可弹反 Hitbox 激活期间，玩家 ParryArea 与该 Hitbox 成功重叠。

序列：
1. 立刻取消扑击剩余流程。
2. 关闭锈犬攻击 Hitbox。
3. 锈犬沿扑击反方向或远离玩家方向弹开约 60px。
4. 进入硬直约 1.0 秒。
5. 硬直结束后重新检测玩家位置。

结束后：若玩家仍在检测范围内，回到 Chase；否则回到 Patrol。

设计理由：弹反成功必须给玩家明确收益：不受伤、获得能量、创造输出窗口。1.0 秒硬直能让玩家打出约 2 次普攻，是“弹反成功 → 反击”的最小闭环。

[受伤]

触发条件：玩家攻击命中锈犬 Hurtbox。

序列：
1. HealthComponent 扣除伤害。
2. 播放 P0 占位受击反馈，例如闪白或短暂变色。
3. 若生命值大于 0，当前行为默认继续。
4. 若生命值小于等于 0，进入死亡。

结束后：继续当前状态或进入 Dead。

设计理由：P0 的锈犬属于节奏教学敌人。普通攻击可以稳定削血，但不应频繁打断敌人动作，否则玩家会倾向于站桩普攻，削弱弹反和闪避学习。

[死亡]

触发条件：生命值小于等于 0。

序列：
1. 取消当前行为。
2. 关闭 Hitbox / Hurtbox / AI 输入。
3. 播放死亡占位反馈。
4. 发出敌人死亡事件。
5. 从战斗中移除，或进入不可交互死亡状态。

结束后：房间系统收到死亡事件，用于清房统计。

设计理由：死亡是敌人最高优先级状态，必须干净关闭所有判定，避免死后继续伤害玩家或继续被房间重复统计。

### 3.2 中断优先级

锈犬中断优先级（从高到低）：

[5-最高] 死亡
- 触发条件：HP ≤ 0。
- 处理：取消所有行为，关闭判定区，发出 `enemy_died(enemy)`。
- 不可被其他状态打断。

[4-高] 被弹反
- 触发条件：可弹反 Hitbox 被玩家 ParryArea 成功判定。
- 处理：取消扑击，关闭 Hitbox，进入 ParriedStun。
- 不保留扑击剩余位移。

[3-中] 攻击流程
- 触发条件：进入 Charge / Lunge / Recovery。
- 处理：攻击流程中不因玩家离开攻击范围而取消。
- 可被死亡、被弹反打断。

[2-低] 受击
- 触发条件：被玩家普攻命中。
- 处理：扣血和播放受击反馈，但默认不打断当前行为。
- 若扣血后死亡，则进入死亡。

[1-最低] 巡逻 / 追击
- 可被攻击流程、受击、被弹反、死亡覆盖。

### 3.3 中断交互

[中断交互] 后蹲蓄力期间玩家离开攻击范围

条件：锈犬已进入后蹲蓄力，玩家随后离开攻击范围。

处理：锈犬不取消本次攻击，继续向已锁定方向扑击。

设计理由：玩家看到前摇后闪避或走位，应获得“骗出攻击”的空间收益。如果锈犬实时追踪并取消蓄力，玩家会很难理解攻击节奏，也不利于学习闪避。

[中断交互] 扑击命中玩家与被弹反接近同时发生

条件：锈犬 Hitbox 与玩家 Hurtbox / ParryArea 在接近同一时刻发生重叠。

处理：若玩家处于有效 ParryArea 且敌方 Hitbox 可弹反，优先按弹反成功处理；否则按受伤处理。具体冲突优先级以玩家行为系统的弹反判定为准。

设计理由：教学怪要鼓励玩家相信弹反。只要玩家在有效窗口内按出弹反，就应优先获得成功反馈，避免“我明明弹了却被咬”的挫败。

[中断交互] 多只锈犬同时存在

条件：P1 或后续房间出现 3-5 只锈犬。

处理：每只锈犬独立计时，不做全局同步。P0 不实现专门编队，只允许自然错开攻击节奏。

设计理由：单只锈犬教弹反；多只锈犬让纯弹反变得不稳定，逼玩家开始使用闪避、走位和超频清理。这是后续组合压力，不应提前复杂化 P0。

## 4. 数值与依据

| 属性 | P0 数值 | 依据 |
|------|--------:|------|
| 生命值 | 40 | 玩家基础普攻 10 伤害时约 4 刀击杀；足够看见一次完整攻击，不会拖慢教学 |
| 攻击力 | 8 | 玩家 100 HP 时约 12-13 次才死亡；新手有容错 |
| 检测范围 | 220px | 测试房间中能主动发现玩家，但不至于全图锁定 |
| 脱战范围 | 320px | 防止玩家刚离开检测边缘就反复切状态 |
| 攻击范围 | 56px | 接近玩家后才蓄力，便于玩家看见扑击起点 |
| 移动速度 | 110px/s | 略慢于玩家，形成追击压力但不给黏人感 |
| 后蹲蓄力 | 0.45 秒 | 前摇清楚，适合作为第一个弹反教学动作 |
| 可弹反窗口 | 0.27 秒 | 宽松教学窗口；实现与验收以秒为准 |
| 扑击距离 | 90px | 能越过短距离威胁玩家，闪避可躲开 |
| 扑击判定时长 | 0.27 秒 | 与可弹反窗口一致，便于 P0 实现和调试 |
| 落地恢复 | 0.20 秒 | 普攻或拉开距离的小窗口 |
| 攻击间隔 | 2.0 秒 | 节奏稳定，玩家可在几轮内形成预判 |
| 被弹反硬直 | 1.0 秒 | 弹反后约可打 2 刀，形成清晰奖励 |
| 弹反弹开距离 | 60px | 拉开安全距离，避免弹反后立刻被贴身 |

数值边界：
- 如果玩家总是被咬中，优先加长后蹲蓄力或增强视觉提示，不先降伤害。
- 如果玩家只靠普攻站桩击杀，优先让扑击更稳定进入威胁范围，不先加血。
- 如果弹反后收益不足，优先增加硬直或弹开表现，谨慎提高回能。

## 5. 与其他系统的联系

**与玩家行为系统**：锈犬是玩家弹反、闪避、受伤、死亡逻辑的第一个真实验证对象。它的扑击必须能被玩家 ParryArea 识别，也必须能在玩家没有弹反或闪避时造成伤害。→ [[策划案_玩家行为系统]]

**与能量系统**：锈犬提供 P0 最稳定的弹反回能机会。玩家弹反锈犬成功后，由玩家行为系统发出 `parry_succeeded(is_perfect)`，能量系统根据完美 / 普通弹反增加能量。锈犬本身不直接修改能量。→ [[策划案_能量系统]]

**与房间推进系统**：锈犬死亡后发出 `enemy_died(enemy)`，房间系统据此统计清怪进度。P0-5 的结晶点出现依赖清房事件，因此锈犬死亡事件必须可靠。

**与打击感反馈**：锈犬被普攻击中、被弹反、死亡都需要可感知反馈。P0-4 可先做闪白 / 变色 / 弹开；音效、HitStop、屏震统一归 P0-7 处理。→ [[策划案_打击感与反馈]]

**与怪物生态**：锈犬是“清晰前摇 + 稳定节奏 + 可弹反攻击”的基准线。后续敌人应从节奏、数量、可弹反关系或空间压力上形成差异，避免只做数值放大。→ [[DESIGN_敌人系统]]

## 6. 美术/音效需求

P0 只要求占位资源能表达状态，不追求正式美术。

### 6.1 美术与动画

| 资源 | P0 需求 | 说明 |
|------|---------|------|
| 占位精灵 | 俯视角犬形轮廓，带金属甲片或锈色区分 | 比玩家略小，读得出“会扑的生物” |
| Idle / Patrol | 简单站立或小幅摆动 | 表达巡逻军犬残留本能 |
| Run / Chase | 低头追击 | 让玩家能判断锈犬正在逼近 |
| Charge | 明显后蹲、压低身体、朝向锁定 | 最关键教学动画，必须清楚 |
| Lunge | 向前扑出 | 和 Hitbox 激活阶段同步 |
| ParriedStun | 弹开、趴伏或短暂僵直 | 明确告诉玩家弹反成功 |
| Hurt | 闪白或短暂变色 | P0 占位反馈 |
| Dead | 消失、碎裂或倒地 | P0 可先用淡出 |

### 6.2 音效

P0-4 不强制实现音频播放，只保留触发点。实际音效播放统一在 P0-7 的反馈系统或 P1 怪物独立音效阶段接入。

| 音效 | 触发点 | 说明 |
|------|--------|------|
| 低吼 / 金属摩擦 | Charge 开始 | 提醒玩家“要扑了” |
| 扑击风声 | Lunge 开始 | 增强动作方向感 |
| 金属弹反声 | 被弹反 | 和玩家弹反反馈配合 |
| 受击声 | Hurt | 可与通用敌人受击音共用 |
| 死亡消散声 | Dead | P1 后可做独立音色 |

## 7. 程序需求

技术方案遵循 [[INDEX_开发路线图#一、Godot 设计哲学速查]]：组合式场景、Resource 配置、信号解耦。实现细节在开发前按当前 Godot 工程验证，策划案只定义结构、职责和验收合同。

### 7.1 场景结构

建议场景：

```text
RustHound (CharacterBody2D)
├── Sprite2D / AnimatedSprite2D
├── CollisionShape2D
├── Hurtbox (Area2D)
├── AttackHitbox (Area2D)
│   └── CollisionShape2D
├── HealthComponent
├── StateMachine
│   ├── PatrolState
│   ├── ChaseState
│   ├── ChargeState
│   ├── LungeState
│   ├── RecoveryState
│   ├── ParriedStunState
│   └── DeadState
└── Debug / Preview 节点（可选）
```

P0 可以把 Patrol 简化为 Idle，但状态边界要保留，避免后续加巡逻路径时重写主体逻辑。

### 7.2 EnemyData 字段

| 字段 | 类型 | P0 值 | 说明 |
|------|------|------:|------|
| `display_name` | StringName | `ENEMY_RUST_HOUND` | 多语言 key |
| `max_health` | int | 40 | 最大生命值 |
| `attack_damage` | int | 8 | 扑击伤害 |
| `move_speed` | float | 110.0 | 追击速度 |
| `detection_range` | float | 220.0 | 发现玩家距离 |
| `lose_target_range` | float | 320.0 | 脱战距离 |
| `attack_range` | float | 56.0 | 进入攻击流程距离 |
| `attack_interval` | float | 2.0 | 两次攻击开始之间的最小间隔 |
| `charge_duration` | float | 0.45 | 后蹲蓄力时间 |
| `lunge_duration` | float | 0.27 | 扑击判定时间 |
| `lunge_distance` | float | 90.0 | 扑击位移 |
| `recovery_duration` | float | 0.20 | 落地恢复 |
| `parryable_duration` | float | 0.27 | 可弹反窗口，单位秒 |
| `parried_stun_duration` | float | 1.0 | 被弹反硬直 |
| `parried_knockback_distance` | float | 60.0 | 被弹反弹开距离 |

说明：`parryable_duration` 与 `lunge_duration` P0 可保持一致，降低实现复杂度。后续若需要“扑击前半段可弹、后半段不可弹”，再拆分字段。

### 7.3 碰撞层需求

沿用玩家行为系统的碰撞层规划：

| Layer | 用途 |
|-------|------|
| Layer 1 | 环境（墙壁） |
| Layer 2 | 玩家 Body |
| Layer 3 | 敌人 Body / Hurtbox |
| Layer 4 | 玩家 Hitbox |
| Layer 5 | 敌人 Hitbox |
| Layer 6 | 玩家 ParryArea |

锈犬需求：
- Body / Hurtbox 属于敌人层，可被玩家 Hitbox 命中。
- AttackHitbox 属于敌人 Hitbox 层，可命中玩家 Hurtbox / Body。
- AttackHitbox 在可弹反阶段需要能被玩家 ParryArea 检测。
- AttackHitbox 激活时标记 `is_parryable = true`，或加入 `parryable_hitbox` group。

### 7.4 信号与事件

| 事件 / 信号 | 参数 | 触发时机 | 用途 |
|-------------|------|----------|------|
| `enemy_died` | enemy: Node | 锈犬死亡 | 房间清怪统计 |
| `enemy_hit_player`（可选） | enemy, damage | 锈犬命中玩家 | 调试或后续反馈系统 |
| `enemy_parried`（可选本地信号） | enemy, is_perfect | 锈犬被弹反 | 锈犬进入硬直与反馈 |

与玩家弹反的对接：
- 玩家 ParryArea 识别锈犬 AttackHitbox 为可弹反目标。
- 若 AttackHitbox 有 `on_parried(is_perfect)`，玩家弹反成功时调用该方法。
- 锈犬收到 `on_parried` 后，关闭 AttackHitbox 并进入 ParriedStun。
- `parry_succeeded(is_perfect)` 仍由玩家行为系统发出，供能量系统监听；锈犬不负责给玩家加能量。

### 7.5 状态机需求

P0 状态最小集合：

| 状态 | 职责 |
|------|------|
| Patrol / Idle | 玩家不在检测范围时待机或巡逻 |
| Chase | 朝玩家移动并判断攻击范围 |
| Charge | 后蹲蓄力，锁定扑击方向 |
| Lunge | 扑击位移，开启 AttackHitbox 和可弹反标记 |
| Recovery | 落地恢复，关闭 AttackHitbox |
| ParriedStun | 被弹反后弹开和硬直 |
| Dead | 关闭 AI 与判定，发死亡事件 |

状态约束：
- Charge 开始后锁定扑击方向，不持续追踪玩家。
- Lunge 期间 AttackHitbox 单次攻击同一玩家只造成一次伤害。
- 进入 ParriedStun / Dead 时必须关闭 AttackHitbox。
- Dead 不可返回其他状态。

### 7.6 当前工程接入点

当前工程已有玩家攻击、弹反和 EventBus 基础。锈犬实现时需要对接：

| 当前能力 | 锈犬需求 |
|----------|----------|
| 玩家 AttackArea 命中有 `take_damage(amount)` 的目标 | 锈犬实现 `take_damage(amount)` 或通过 Hurtbox 转发到 HealthComponent |
| 玩家 ParryArea 会识别 `parryable_hitbox` group 或 `is_parryable` 属性 | 锈犬 AttackHitbox 在 Lunge 期间加入 group 或设置属性 |
| 玩家弹反成功会调用目标 `on_parried(is_perfect)` | 锈犬 AttackHitbox 实现该方法并通知锈犬本体 |
| EventBus 已有 `enemy_died(enemy)` | 锈犬死亡时发出该事件 |

## 8. 验收标准

- [ ] 锈犬使用独立场景，基于 CharacterBody2D 或等价敌人根节点，包含 HealthComponent、Hurtbox、AttackHitbox、StateMachine。
- [ ] 锈犬能在检测范围内发现玩家，并进入追击。
- [ ] 锈犬进入攻击范围后执行后蹲蓄力，蓄力表现清楚可见。
- [ ] 后蹲蓄力约 0.45 秒后，锈犬向锁定方向扑击。
- [ ] 扑击阶段敌方 AttackHitbox 激活，命中玩家造成 8 点伤害。
- [ ] 扑击阶段 AttackHitbox 可被玩家 ParryArea 识别为可弹反。
- [ ] 玩家在有效弹反窗口内弹反成功后，锈犬取消扑击，关闭 Hitbox，进入约 1.0 秒硬直。
- [ ] 弹反成功后玩家能通过 `parry_succeeded(is_perfect)` 获得能量，锈犬本身不直接改能量。
- [ ] 玩家普攻命中锈犬会扣除锈犬生命值，并播放 P0 占位受击反馈。
- [ ] 锈犬生命值归零后关闭判定并发出 `enemy_died(enemy)`。
- [ ] 死亡后的锈犬不会继续移动、攻击、被重复计入清怪。
- [ ] 相关时间实现以秒为准，不使用画面刷新次数作为判定主单位。

## 10. 待验证问题

1. 0.45 秒后蹲蓄力是否足够清楚，是否需要更夸张的视觉预警。
2. 0.27 秒可弹反窗口是否太宽，是否会让锈犬长期失去威胁。
3. 2.0 秒攻击间隔在单只锈犬场景下是否太慢，是否导致教学房间节奏空。
4. 40 HP 是否能让玩家至少看到一次扑击；若玩家超频普攻很早成型，是否会被秒杀得太快。
5. 普攻击中不打断锈犬是否会让玩家感到“砍了没反馈”；P0 是否需要轻微受击停顿。
6. 多只锈犬自然错开攻击是否足够制造压力，还是需要房间系统做轻量节奏调度。
