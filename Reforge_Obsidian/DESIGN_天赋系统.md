## 一、系统定位

天赋系统是 Reforge 的局内构筑核心。它把光核与祭坛提供的成长机会转化为实际玩法变化，让玩家从前期的基础动作循环进入中后期的 Build 成型。

它位于 [[DESIGN_光核与祭坛]] 和 [[DESIGN_玩家行为系统]] 之间：

```text
战斗表现 → 能量保留 → 光核结晶 → 祭坛兑换 → 天赋选择 → 动作规则变化 → 新的战斗表现
```

玩家行为系统提供可被改写的动作入口，能量系统提供资源张力，光核与祭坛提供天赋获取节奏，天赋系统负责让这些输入形成具体构筑。

如果没有天赋系统，Reforge 只剩弹反、超频和结晶的资源循环；有了天赋系统，玩家每局都能形成不同的动作节奏、能量来源和战斗解法。

---

## 二、设计目标

### 2.1 让构筑成为核心卖点

Reforge 的长期乐趣来自“这局我要玩什么 Build”。天赋系统需要让玩家在一局内逐步形成方向，从最初的动作强化，走向规则改造，最终获得定义玩法的核心循环。

玩家在前期通过基础动作理解系统，在中期通过天赋改变能量输入和动作关系，在后期通过核心天赋获得明显的成型感。

### 2.2 让选择带有可读方向

每次天赋选择都应让玩家看懂它指向什么玩法。玩家不需要提前记住完整天赋池，也能通过天赋描述判断：这个选择是在强化弹反、普攻、闪避、技能、超频，还是在连接两个动作。

方向可读的天赋能降低构筑门槛。玩家可以凭直觉选择，也可以在熟悉系统后进行更深的组合规划。

### 2.3 让组合产生涌现空间

天赋系统要提供预设路线，也要允许跨路线组合。五条流派骨架提供清晰道路，桥梁天赋提供岔路口，中性天赋提供底层规则变化。

玩家理想中的体验是：前几次选择看见方向，中期发现组合，后期感到“这套玩法是我拼出来的”。

### 2.4 允许强力构筑出现

构筑游戏需要允许玩家获得明显超规格的组合。强力构筑会制造惊喜、复玩动力和分享价值。

平衡目标是维持多路线可行性。某个 Build 可以在一局中爆发，长期环境中需要避免单一组合压过其他路线。

### 2.5 保留稳定性工具

天赋池需要同时提供伤害、爆发和稳定性工具。稳定性天赋负责降低循环断裂、资源卡手和输入负担。

稳定性天赋包括：改善能量回收、减少超频输入压力、提高动作容错、优化天赋选择质量、让关键动作更容易接入循环。

## 三、核心规则

### 3.1 三选一获取

玩家通过祭坛消耗完整光核，触发一次天赋选择。每次选择从随机生成的三个天赋中选一个。

三选一结构承担三件事：

- 保留 Roguelike 的随机性。
- 让玩家在有限选项中做取舍。
- 让每局 Build 路径具有差异。

天赋是局内资源。每局开始时，天赋重新获取；跨局永久升级属于 Meta 成长系统。

### 3.2 天赋围绕动作设计

天赋以玩家动作为主要挂点，包括：

- 普通攻击
- 弹反
- 闪避
- 超频
- 技能

动作是构筑的输入端。天赋通过强化、改造或连接动作，让玩家形成不同的战斗循环。

流派由天赋组合自然形成。设计文档中的五条路线是构筑骨架，用来保证玩家能看见方向。

### 3.3 天赋分层递进

天赋分为三级：

| 层级 | 作用 | 设计重点 |
|------|------|----------|
| 一级 · 强化层 | 让某个动作更好用 | 方向暗示、低理解成本、轻量数值变化 |
| 二级 · 改造层 | 改变动作规则或动作关系 | 新能量输入、动作联动、桥梁组合 |
| 三级 · 核心层 | 定义 Build 循环并给予技能 | 玩法质变、强成型感、割草入口 |

三层结构让玩家从“我这个动作变强了”逐步过渡到“我这套循环成立了”。

### 3.4 天赋按功能分类

天赋同时拥有层级和功能类型。层级说明它在局内成长中的阶段，功能类型说明它对构筑结构的作用。

| 类型    | 作用            | 例子                  |
| ----- | ------------- | ------------------- |
| 入口天赋  | 给玩家方向暗示       | 弹反窗口增加、普攻强化         |
| 引擎天赋  | 增加或改变能量输入     | 普攻命中回能、闪避后短时间弹反回能提高 |
| 消耗天赋  | 改变能量出口        | 超频消耗降低、超频击杀返还       |
| 桥梁天赋  | 连接两个动作        | 弹反成功后下一次普攻强化        |
| 稳定性天赋 | 降低循环断裂和输入负担   | 下一次超频动作标识、超频锁定候选规则  |
| 核心天赋  | 定义完整 Build 循环 | 弹反进入子弹时间、普攻标记爆发     |

一个天赋可以同时拥有多个功能标签。例如“弹反成功后，下一次动作自动视为超频动作”既是桥梁天赋，也是消耗相关天赋。

### 3.5 天赋有获取条件

高层天赋需要一定前置关系，避免玩家在完全没有基础循环时拿到难以使用的核心规则。

基础原则：

- 一级天赋可以较早出现，用于给方向。
- 二级天赋需要与已有动作方向或通用规则产生关系。
- 三级天赋需要玩家已经拥有对应方向的基础与改造天赋。
- 中性天赋可以跨流派出现，用于改善底层规则或选择质量。

获取条件服务构筑连贯性。它让随机选项保持变化，同时避免选择结果过散。

## 四、构筑原则

### 4.1 方向可见

玩家看到一个天赋时，应能判断它在支持哪类玩法。天赋名称、描述、图标和数值都要传达方向。

方向可见允许天赋拥有隐藏深度。玩家先理解“这个选择想让我做什么”，再逐渐理解“它还能和哪些天赋组合”。

### 4.2 复杂度分层

一级天赋保持直观。二级天赋开始改变动作关系。三级天赋定义循环。

这种分层让新手可以先按动作偏好选择，熟练玩家可以从二级开始规划组合，高手可以围绕三级核心反推整套 Build。

### 4.3 每个动作都能成为构筑入口

普攻、弹反、闪避、超频都需要拥有可见的成长路线。玩家可以从任一动作进入构筑，再通过桥梁天赋接到其他动作。

弹反承担前期能量启动职责。后期构筑需要让普攻、闪避、技能和超频都获得新的能量输入或输出价值。

### 4.4 桥梁天赋制造混搭

桥梁天赋连接两个动作，使玩家愿意从当前路线转向另一个动作维度。桥梁天赋是混搭体验的关键。

桥梁天赋需要让玩家产生明确联想：我已经有 A，如果再拿 B，这个循环会更完整。

桥梁天赋占天赋池约 20-30%。比例过低会让流派过窄，比例过高会让路线方向变模糊。

### 4.5 稳定性天赋支撑循环

构筑的爽感依赖稳定循环。天赋池需要提供一些不直接制造爆发、但能让 Build 更顺的选择。

稳定性天赋可以处理：

- 能量输入不稳定。
- 超频动作高频使用带来的输入负担。
- 关键动作窗口过窄。
- 天赋选择过散。
- 技能或核心循环启动太慢。

稳定性天赋的价值不一定体现在单次伤害，而体现在整局 Build 的可持续性。

### 4.6 强力组合要有代价

强力组合可以存在，但需要带有资源、条件、时机或风险成本。

常见代价包括：

- 消耗更多能量。
- 需要连续动作成立。
- 需要击杀或完美时机触发。
- 短时间内降低其他收益。
- 依赖特定前置天赋。

代价让强力组合成为需要经营的构筑目标。

## 五、天赋层级

### 5.1 一级 · 强化层

一级天赋负责给方向。它们通常强化单个动作，降低玩家尝试路线的门槛。

一级天赋适合：

- 增加动作容错。
- 提高基础数值。
- 降低能量消耗。
- 提供轻量触发效果。

示例：

- 弹反窗口增加。
- 普攻伤害提高。
- 闪避距离增加。
- 超频普攻消耗降低。

一级天赋应避免过早改变大量规则。它的职责是让玩家看见方向，并愿意继续沿着该方向选择。

### 5.2 二级 · 改造层

二级天赋负责改变动作关系。它让构筑从“某个动作更强”进入“动作之间开始循环”。

二级天赋适合：

- 让动作之间产生触发链。
- 增加新的能量输入方式。
- 改变某个动作的使用时机。
- 提供跨流派桥梁。

示例：

- 普攻命中累积标记，标记满后下一次动作触发爆炸。
- 闪避后短时间内弹反窗口增加。
- 弹反成功后，下一次动作视为超频动作。
- 超频闪避留下残影，对路径敌人造成伤害。

二级天赋是 Build 开始成型的转折点。玩家从这里开始改变战斗打法。

### 5.3 三级 · 核心层

三级天赋负责定义玩法循环。它是强力的 Build-defining 天赋，拿到后玩家的主要战斗目标和操作节奏会发生明显变化。

三级天赋适合：

- 建立完整循环。
- 明确当前 Build 的胜利条件。
- 给予技能或技能化效果。
- 让玩家获得中后期割草感。

示例：

- 完美弹反后进入短暂子弹时间，期间动作回复更多能量。
- 连续普攻触发剑气风暴，风暴中闪避刷新普攻计数。
- 超频动作消耗降低，连续击杀返还部分能量。

三级天赋的强度可以明显高于一级和二级。它代表构筑成果，需要通过前置关系和能量循环承担代价。

## 六、五条流派骨架

### 6.1 弹反流

弹反流以敌人攻击为资源机会。玩家通过准确判断前摇，获得能量、反击窗口和后续强化。

核心体验：

- 精准反制。
- 高回报操作。
- 利用敌人攻击反过来推动 Build。

设计方向：

- 提高弹反窗口或奖励。
- 弹反后触发冲击波、反击、回能或子弹时间。
- 将弹反结果连接到普攻、超频或技能。

### 6.2 普攻流

普攻流以持续输出和连段积累为核心。它让玩家通过主动进攻建立标记、层数或节奏。

核心体验：

- 连击节奏。
- 贴身压制。
- 从稳定输出滚雪球到范围爆发。

设计方向：

- 普攻连段强化。
- 命中叠层或标记。
- 普攻触发能量、爆炸、技能冷却或超频收益。

### 6.3 闪避流

闪避流以位置、时机和风险回避为核心。它让防御行为成为进攻或能量循环的一部分。

核心体验：

- 极限回避。
- 灵活穿插。
- 用位置优势换输出机会。

设计方向：

- 完美闪避奖励。
- 闪避后强化下一次攻击或弹反。
- 闪避路径造成伤害、留下残影或改变敌人状态。

### 6.4 技能流

技能流以三级天赋赋予的技能为核心。技能是构筑成果，代表 Build 成型后的新能力。

核心体验：

- Build 成型后的质变。
- 明确的强力释放时刻。
- 从动作循环进入技能循环。

设计方向：

- 技能由三级天赋赋予。
- 技能冷却、充能或触发条件与动作循环绑定。
- 技能可以反过来影响能量获取、光核节奏或超频收益。

### 6.5 超频流

超频流以高频强化动作和能量自循环为核心。玩家通过消耗能量获得即时强度，再通过天赋返还或降低成本维持循环。

核心体验：

- 高频强化。
- 资源赌注。
- 用当前战斗收益换取持续压制。

设计方向：

- 降低特定超频动作消耗。
- 超频击杀返还能量。
- 连续超频动作叠加收益。
- 解锁降低输入负担的稳定性规则，例如超频锁定候选机制。

## 七、资源流 / 行为流

天赋系统的构筑流：

```text
战斗行为产生能量
→ 玩家选择消耗或保留能量
→ 战后结晶形成光核
→ 祭坛消耗光核
→ 生成三选一天赋
→ 玩家选择天赋
→ 天赋改写动作、能量或技能规则
→ 新规则影响下一轮战斗行为
```

天赋系统的核心闭环：

```text
动作表现 → 资源积累 → 构筑选择 → 动作变化 → 更高效的动作表现
```

天赋需要回到动作、能量、超频、技能或光核节奏中，改变玩家下一场战斗的选择。

## 八、关键数值与原则

| 项目 | 原则 |
|------|------|
| 每次选择数量 | 三选一，保证取舍清晰 |
| 一局天赋总量 | 约 10-14 个，支撑一条主线加少量混搭 |
| 天赋层级 | 一级给方向，二级改规则，三级定循环 |
| 桥梁天赋比例 | 约 20-30%，支撑跨动作混搭 |
| 中性天赋比例 | 约 15-20%，提供底层规则变化和稳定性 |
| 强力核心天赋 | 允许明显超规格，但需要条件或代价 |

天赋数值的核心原则：

- 一级天赋优先保证可读性和方向感。
- 二级天赋优先制造动作关系。
- 三级天赋优先制造质变和成型感。
- 稳定性天赋可以低伤害，但要明显改善循环顺畅度。
- 同一路线内需要存在多个可行分支，避免单一最优答案长期固定。

## 九、系统边界

天赋系统负责：

- 定义局内天赋的获取方式。
- 定义天赋层级、功能类型和流派骨架。
- 决定天赋如何改写动作、能量、超频或技能规则。
- 管理天赋拥有权与天赋运行时状态的生命周期。
- 提供 Build 成型的节奏和方向。
- 为玩家提供可读的构筑选择。

天赋系统不负责：

- 光核的生成与消耗结算。
- 玩家基础动作的原始输入与状态机。
- 敌人 AI 和房间生成。
- 跨局永久升级。
- 具体 UI 布局、按钮、图标资源和程序实现细节。

---

## 十、状态生命周期原则

天赋系统需要同时处理两类状态：

| 类型 | 含义 | 示例 |
|------|------|------|
| 天赋拥有权 | 玩家本局是否获得某个天赋 | 已获得裂纹斩、护势格挡、余热回流 |
| 天赋运行时状态 | 天赋在当前战斗、房间或选择流程中留下的临时结果 | 敌人身上的裂纹层数、玩家当前护势层数、当前三选一候选 |

天赋拥有权决定“玩家具备哪些规则改写资格”。天赋运行时状态决定“这些规则在当前时刻留下了什么结果”。两者需要分别记录，也需要同步清理。

当本局重置、天赋被移除、角色死亡、房间切换或系统明确要求清理时，对应的运行时状态也必须清理。这样可以避免玩家已经失去某个天赋，但该天赋产生的护势、裂纹、回流标记等效果继续残留。

后续新增 Buff、叠层、标记、召唤物、临时强化或选择候选时，都应定义：

- 何时获得。
- 何时消耗。
- 何时随天赋拥有权消失。
- 何时随新一局、房间切换或死亡清空。

## 十一、与其他系统的关系

### 11.1 与玩家行为系统

[[DESIGN_玩家行为系统]] 提供天赋可改写的动作入口。普通攻击、弹反、闪避、超频和技能槽都需要暴露清晰的触发时机，让天赋可以稳定接入。

天赋系统通过改变这些动作的数值、触发关系和后续效果，让玩家行为逐渐从基础动作变成 Build 循环。

### 11.2 与能量系统

[[DESIGN_能量系统]] 是天赋系统最重要的资源基础。天赋可以改变能量获取方式、能量消耗效率、超频动作收益和能量循环稳定性。

能量系统提供“生存 vs 成长”的基础张力，天赋系统负责让这条张力在不同流派中出现不同解法。

### 11.3 与光核与祭坛

[[DESIGN_光核与祭坛]] 提供天赋获取入口。光核决定玩家何时能获得天赋，祭坛决定天赋选择发生的位置和节奏。

天赋系统接收祭坛触发的选择机会，并将其转化为局内玩法变化。

### 11.4 与敌人系统

[[DESIGN_敌人系统]] 提供天赋价值的验证环境。不同敌人攻击节奏、数量密度和可弹反机会，会让不同天赋表现出差异。

敌人设计需要支持多种动作路线：有些敌人适合弹反，有些敌人适合普攻清理，有些敌人适合闪避拉扯或技能爆发。

### 11.5 与关卡结构

关卡结构决定玩家获得天赋的节奏。祭坛节点、战斗房间、奖励预览、商店和事件共同影响 Build 成型速度。

天赋系统需要和房间推进节奏配合，让玩家在一局中经历方向出现、循环建立、核心成型三个阶段。

### 11.6 与 Meta 成长系统

Meta 成长系统提升底层能力和长期舒适度。天赋系统负责局内规则变化。

Meta 可以提升光核上限、基础生命、基础能量池或选择质量；天赋负责改变动作循环、能量关系和技能获得。

## 十二、世界观 / 表现对应

天赋在叙事上对应主角身体被异力逐步改造的过程。前期名称可以使用“神恩”“净化”“祝福”等主角视角词汇，后期揭示其真实含义：这些能力来自异力改造，与骑士团的异化同源。

天赋层级与异化程度对应：

| 天赋层级 | 主角视角 | 真实含义 |
|----------|----------|----------|
| 一级强化层 | 神恩增强 | 身体出现轻微改造 |
| 二级改造层 | 神恩显现 | 异力开始改变动作规则 |
| 三级核心层 | 神之器皿 | 身体进入接近异端的状态 |

天赋表现需要体现“变强”和“异化”同时发生。早期反馈可以偏圣洁，后期反馈逐渐转向异力、锻造、身体改写和失控边缘。

具体命名切换规则见 [[DESIGN_世界观设计文档#3.2 双层命名切换系统]]。

## 十三、设计风险与待验证问题

1. 一级天赋是否足够有趣，能否在不改变规则的情况下给玩家方向感。
2. 二级天赋的动作联动是否足够清晰，玩家能否理解组合价值。
3. 三级天赋是否能带来明显成型感，同时保持代价和条件。
4. 桥梁天赋比例是否能支撑混搭，且不会让流派方向变得模糊。
5. 超频流是否会因为高频按键产生操作负担，需要稳定性天赋缓解。
6. 天赋池是否会出现单一最优路线，导致玩家每局都追同一个答案。
7. 稳定性天赋是否会被玩家低估，需要通过描述、图标和实际体验体现价值。
8. 技能作为三级天赋成果是否足够有成就感，是否会让前期技能槽空置显得奇怪。
9. 天赋与敌人配置是否能互相验证，让不同 Build 在不同房间中各有优势。
10. 强力组合出现时，玩家是否感到“我构筑成功了”，而非系统失控或数值失衡。
