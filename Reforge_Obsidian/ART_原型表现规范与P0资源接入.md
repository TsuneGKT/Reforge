## 一、定位

本文件定义 P0 后半段的原型表现标准。目标是让游戏从“功能能跑”推进到“状态可读、手感可判断、流程可展示”。

P0 采用原型资产迭代。角色、敌人、武器、特效、地图和音效都以可读性为第一目标。正式美术风格、完整动画资源和统一视觉包装归后续阶段。

P0 原型表现需要回答三个问题：

1. 玩家能否看出自己当前处于什么动作状态。
2. 玩家能否看出敌人当前处于什么行为状态。
3. 玩家能否通过视觉和音效判断命中、弹反、受伤、超频、结晶、祭坛和天赋选择已经成立。

---

## 二、世界观边界

P0 原型表现从现有世界观中取可执行信息：

- 边境骑士团废墟。
- 异力侵蚀。
- 锈蚀金属、灰石残垣、荒野和堡垒遗迹。
- 主角前期以为自己在使用净化能力，真相是身体兼容异力。
- 锈犬是骑士团军犬死后被异力驱动，犬甲与躯体融合。
- 结晶点和祭坛是战斗资源转为构筑成长的场景物件。

P0 原型资产只需要表达这些信息的基本方向：锈蚀、废墟、异力光、武器动作、结晶物件和祭坛交互。完整区域风格和正式角色设计归后续美术阶段。

## 三、基准单位

### 3.1 Tile 与分辨率

| 项目 | P0 基准 |
|------|---------|
| 逻辑 tile | 16 × 16 px |
| 显示倍率 | Godot 窗口可整数倍放大，保持 nearest 像素采样 |
| 角色视觉高度 | 玩家约 2 tile，高 32 px |
| 玩家脚底碰撞 | 约 0.75-1 tile，围绕脚底 / 身体中心 |
| 锈犬视觉尺寸 | 长约 2 tile，高约 1-1.5 tile |
| 武器攻击距离 | 约 1.5-2 tile |
| 推荐视口 | 640 × 360 px |
| P0 原型房间总尺寸 | 44 × 26 tile，约 704 × 416 px |
| 可战斗区域 | 约 36 × 20 tile，约 576 × 320 px |
| 中央核心战斗区 | 约 28 × 14 tile，约 448 × 224 px |

逻辑单位以 16 px 为准。导入 32 px 或 64 px 地图素材时，在 TileSet 或场景层保持统一比例，让玩家、敌人和判定继续沿用 16 px 逻辑尺寸。

P0 原型房间参考《哈迪斯》式单房间战斗空间：中央留出开阔战斗区，边缘承载墙体、装饰、结晶点、祭坛和出口。调试场景可以更小，完整 P0 验收房间按上表尺寸制作。

### 3.2 俯视 3/4 表现

P0 使用 2D 俯视 3/4 表现：

- 地面按 TileMap 绘制。
- 角色和敌人使用能看到头身轮廓的 3/4 俯视轮廓。
- 角色脚底或身体中心作为位置基准。
- 场景物体按 y 轴排序，脚底更靠下的对象显示在前。
- 墙体、祭坛、结晶点等物件需要有清楚底座，便于玩家判断碰撞位置。

## 四、玩家状态原型表现

玩家状态机需要有对应的可视化表现。P0 采用 Line2D 火柴人动作原型：身体、四肢、武器、弧光和反馈效果拆成独立节点，由状态机事件驱动姿态变化。

当前实现方案为方案 A：固定 Line2D 火柴人。玩家由头部、躯干线、左右手臂线、左右腿线、武器线、攻击弧光和弹反格挡弧组成。方案 B：关节点骨架火柴人保留为后续升级方向，用于更细的姿态插值和动画复用。

| 状态 / 事件 | 原型表现 |
|-------------|----------|
| Idle | 面向最后输入方向，武器收在身体侧前方 |
| Move | 身体轻微上下浮动或脚步帧切换，朝向跟随移动方向 |
| Attack 起手 | 武器后摆，身体朝攻击方向锁定 |
| Attack 生效 | 前方出现武器弧光，弧光覆盖攻击判定范围 |
| Attack 收束 | 弧光淡出，武器回到收势位置 |
| Parry | 武器横挡或前方出现格挡弧，颜色使用金色或蓝白色 |
| Parry 成功 | 瞬间闪光、短暂停顿、弹反音和轻微屏震 |
| Dodge | 位移方向出现残影或短拖尾 |
| Hurt | 玩家闪红，发生轻微击退 |
| Dead | 输入停止，线条向外散开并淡出 |
| Overclock 动作 | 当前动作叠加金色轮廓、短脉冲或更亮弧光 |

玩家攻击弧光需要和实际 Hitbox 对齐。调试时允许显示判定框，验收时以弧光作为玩家读懂攻击范围的主要依据。

## 五、锈犬状态原型表现

锈犬是 P0 的敌人状态机展示基准。P0 同样采用 Line2D 火柴原型，使用身体线、头部形状、前后腿线、装甲片、扑击方向线和攻击预警表达状态。

| 状态 / 事件 | 原型表现 |
|-------------|----------|
| Idle / Patrol | 低头待机或短距离巡逻，保留军犬轮廓 |
| Chase | 身体前倾，朝玩家移动 |
| Charge | 身体后蹲蓄力，前方显示扑击方向预警 |
| Lunge | 身体拉伸或拖影，沿锁定方向扑出 |
| ParriedStun | 被弹开、趴伏或短暂僵直，颜色偏蓝白 / 金色闪光 |
| Hurt | 闪白或短暂变亮 |
| Dead | 线条向外散开并淡出，关闭判定 |

锈犬 Charge 和 Lunge 是 P0 最重要的敌人表现。玩家需要能在 Charge 阶段读到“即将扑击”，在 Lunge 阶段判断弹反或闪避时机。

## 六、反馈层实现方向

P0 采用独立反馈层承接表现。玩家状态机和敌人状态机负责规则与事件，反馈层负责音效、屏震和特效。

建议结构：

```text
状态机 / 战斗事件
→ EventBus 或本地信号
→ FeedbackSystem
→ HitStop / ScreenShake / SFX / Flash / Trail / Arc
```

建议节点职责：

| 节点 / 资源 | 职责 |
|-------------|------|
| `PrototypeVisualController` | 管理玩家或敌人的占位身体、武器、弧光、残影、闪烁 |
| `FeedbackSystem` | 统一监听关键事件并触发反馈 |
| `HitStopController` | 管理短暂停顿 |
| `ScreenShakeController` | 管理 Camera2D 微震 |
| `SFXPlayer` | 播放一次性音效 |
| `SoundBank` Resource | 保存事件到音效资源的映射 |

P0 先覆盖这些事件：

| 事件 | 反馈组合 |
|------|----------|
| 普攻命中 | 命中音、敌人闪白、轻 HitStop |
| 完美弹反 | 金属弹反音、强 HitStop、轻屏震、闪光 |
| 玩家受伤 | 受伤音、闪红、轻击退、轻屏震 |
| 超频动作触发 | 金色脉冲、强化音效占位 |
| 结晶成功 | 结晶音、光核 HUD 反馈 |
| 祭坛激活 | 交互确认音、祭坛亮起 |
| 天赋确认 | UI 确认音、卡牌选择反馈 |

## 七、P0 音效资源接入

P0 音效采用授权清楚的社区资源。优先使用 CC0 资源；使用其他免费授权时，需要单独记录来源和限制。

| 类别 | 资源候选 | 用途 |
|------|----------|------|
| UI 音效 | Kenney UI Audio（CC0） | 天赋选择、按钮确认、菜单反馈 |
| 撞击 / 命中 | Kenney Impact Sounds（CC0） | 普攻命中、受伤、轻碰撞 |
| 金属弹反 | Freesound Sword sounds compilation（页面标注 CC0） | 弹反、武器碰撞 |
| 地图 Tile | OpenGameArt Top Down Dungeon Pack（CC0） | 原型房间地面、墙体 |
| 16px 地图备选 | OpenGameArt Deepnight Dungeon Tileset（CC0，16×16） | 更接近 16px 逻辑 tile 的房间原型 |

音效最低接入清单：

- 弹反成功。
- 普攻命中。
- 玩家受伤。
- 锈犬蓄力 / 扑击。
- 结晶成功。
- 祭坛激活。
- 天赋确认。

素材导入建议：

```text
Reforge_Godot/assets/audio/sfx/combat/
Reforge_Godot/assets/audio/sfx/ui/
Reforge_Godot/assets/audio/sfx/interact/
Reforge_Godot/assets/tilesets/prototype/
```

## 八、P0 地图与背景资源接入

P0 地图先做一个完整原型房间：

- 总尺寸约 44 × 26 tile 的单房间战斗空间。
- 约 36 × 20 tile 的可战斗区域。
- 约 28 × 14 tile 的中央核心战斗区。
- 1 个玩家出生点。
- 1-3 只锈犬出生点。
- 结晶点位置。
- 祭坛位置。
- 墙体碰撞。
- 可读的出入口或边界。

地图目标是支持完整流程：

```text
进入房间 → 战斗 → 清房 → 结晶 → 祭坛 → 选择天赋 → 天赋生效
```

地图原型重点：

- 地面和墙体清晰区分。
- 可行走区域边界明确。
- 结晶点和祭坛与普通地面有视觉差异。
- 战斗空间足够让锈犬蓄力、扑击、弹反和闪避发生。
- 摄像头能在默认视角下同时看到玩家、锈犬和关键交互物。

## 九、验收标准

- [ ] 一个 tile、角色高度、碰撞脚底和攻击距离在 Godot 中保持统一。
- [ ] 玩家 Idle / Move / Attack / Parry / Dodge / Hurt / Overclock 都有可识别原型表现。
- [ ] 普攻生效阶段有武器弧光，弧光与攻击判定方向一致。
- [ ] 弹反阶段有武器格挡或格挡弧表现。
- [ ] 锈犬 Chase / Charge / Lunge / ParriedStun / Hurt / Dead 都有可识别原型表现。
- [ ] 玩家和锈犬死亡时都使用线条向外散开并淡出的统一表现。
- [ ] 完美弹反、普攻命中、玩家受伤、超频动作触发有基础反馈组合。
- [ ] 弹反、命中、受伤、结晶、祭坛、天赋确认至少接入占位音效。
- [ ] 测试房间使用现成 tile / 背景资源完成可行走空间。
- [ ] 完整流程可在一个房间中跑通，并能通过视觉与音效判断关键状态和结果。
