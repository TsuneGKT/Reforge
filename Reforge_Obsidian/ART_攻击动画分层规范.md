## 一、用途

本文记录 Reforge 玩家攻击动画的 P0 生产规则。目标是先完成一套可验证的右下攻击样例，把角色动作、武器弧光和攻击判定在 Godot 中对齐，再扩展到移动、弹反、闪避和其他方向。

P0 攻击动画采用分层接入：

```text
角色本体动画
武器弧光 / 攻击特效
攻击 Hitbox
```

角色本体负责姿势和发力，弧光负责攻击范围和方向，Hitbox 负责真实命中。三者在攻击生效帧对齐。

---

## 二、首个验证样例

第一套动画只做玩家右下方向普攻。

| 项目 | 要求 |
|------|------|
| 方向 | 右下 3/4 俯视 |
| 动作 | Idle → 起手 → 挥出 → 收招 |
| 角色画布 | 512 × 512 透明画布 |
| 角色主体 | 保持约 320-380 px 高 |
| 弧光画布 | 512 × 512 透明画布，中心和角色脚底锚点规则一致 |
| Godot 显示 | 使用当前玩家显示目标，主体约 135-145 px 高 |
| 验证场景 | 玩家攻击时角色动作、弧光和 Hitbox 方向一致 |

这个样例通过后，再决定是否补左下镜像、右上 / 左上，以及 Idle / Move 的方向组。

## 三、生产流程

当前攻击动画按以下流程推进：

```text
Codex / GPT 生攻击首帧、关键姿势或参考图
→ 用户把参考图交给即梦生成动作视频
→ 从视频中抽取起手、挥出、收招关键帧
→ PS 清理角色本体，统一轮廓、色块、脚底锚点
→ 单独整理弧光透明 PNG 或 Godot 临时弧光
→ Aseprite 合成角色 Sprite Sheet
→ Godot 中用 AnimationPlayer / AnimatedSprite2D 验证
```

生图阶段允许参考图带弧光，用来确认挥砍力度和画面感觉。正式接入时优先保留可独立控制的弧光层。

---

## 四、角色本体层

角色本体层只表达身体姿势、武器握持和发力方向。

| 阶段 | 表现重点 |
|------|----------|
| 起手 | 武器后摆，身体重心轻微压低，肩部和胸口朝攻击方向锁定 |
| 挥出 | 武器经过身体前方，姿势清楚，脚底锚点稳定 |
| 收招 | 武器回到身体侧前方，身体回到可继续行动的姿态 |

角色本体帧需要保持同一画布尺寸、同一主体大小和同一脚底锚点。武器可以有明显运动姿态，但角色体型不能在帧间跳变。

---

## 五、弧光与特效层

弧光层表达攻击范围、方向和力度。P0 可以先使用 Godot 临时弧光；如果 AI 图中已有合适弧光，再整理为透明 PNG 序列。

| 阶段 | 弧光要求 |
|------|----------|
| 起手 | 不显示或只显示很弱的预备光 |
| 生效 | 出现主要弧光，覆盖攻击 Hitbox 的视觉范围 |
| 收招 | 弧光快速淡出，避免遮挡角色状态 |

弧光颜色、亮度和尺寸后续可服务超频攻击、天赋强化和命中反馈。弧光层应尽量独立，便于旋转、换色、缩放和替换。

---

## 六、Godot 对齐规则

接入 Godot 时，攻击动画至少对齐三件事：

```text
角色挥出关键帧
弧光出现帧
Hitbox 开启帧
```

检查重点：

1. 玩家按攻击键后，起手、挥出、收招节奏是否清楚。
2. 弧光方向是否跟输入方向一致。
3. 弧光覆盖范围是否和实际 Hitbox 接近。
4. 命中反馈是否发生在弧光最亮或最完整的阶段。
5. 角色脚底是否在动画中漂移。

---

## 七、当前待验证

1. 即梦生成的攻击视频能否稳定保持角色大小和脚底锚点。
2. 从视频抽帧后，弧光是否能清理成独立层。
3. 512 × 512 画布是否足够容纳右下挥砍大动作。
4. Godot 临时弧光和 AI 弧光哪一种更适合 P0 展示。
