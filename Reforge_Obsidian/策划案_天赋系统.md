## 1. 设计目的

P0-6 的目标是把“祭坛请求天赋选择”接成真实的局内构筑闭环。

当前 P0-5 已经完成：

```text
清房 → 结晶点互动 → 能量转为光核 → 祭坛消耗光核 → 发出天赋选择请求
```

P0-6 需要补上后半段：

```text
天赋选择请求 → 三选一 UI → 玩家选择天赋 → 天赋记录进 RunData → 天赋效果生效
```

完成后，玩家在一个测试房间中完成“花掉光核 → 获得 Build 成长 → 后续战斗动作规则发生变化”的闭环体验。

P0-6 只验证最小天赋系统：

- 只做 3 个一级候选天赋。
- 只做一次三选一。
- 只验证天赋能被选择、记录和生效。
- 不展开完整天赋池、二级/三级天赋、技能系统、权重池和复杂前置条件。

---

## 2. 系统定位

天赋系统位于 [[DESIGN_光核与祭坛]] 之后，是光核消费后的玩法变化层。

P0-6 中的系统关系：

```text
Altar
→ EventBus.talent_selection_requested
→ TalentSelectionUI
→ TalentSystem
→ RunData.acquired_talents
→ Player / Enemy / OverclockSystem 相关效果生效
```

职责划分：

| 模块 | 职责 |
|------|------|
| Altar | 消耗光核，发出天赋选择请求 |
| TalentSelectionUI | 展示三张天赋卡，接收玩家选择 |
| TalentSystem | 提供候选天赋、应用天赋效果、管理运行期天赋状态 |
| RunData | 保存本局已获得天赋 |
| EventBus | 转发天赋选择、应用和反馈事件 |
| Player / Enemy / OverclockSystem | 承接具体效果 |

## 3. P0-6 天赋池

P0-6 使用 3 个一级天赋，分别验证普攻、弹反、超频三个方向。

| 天赋 | 来源 | 验证目标 |
|------|------|----------|
| 裂纹斩 | [[DATA_天赋池#裂纹斩]] | 普攻命中后给敌人叠加状态，并在层数满时触发额外伤害 |
| 护势格挡 | [[DATA_天赋池#护势格挡]] | 完美弹反后获得防护，下一次受伤时减少伤害 |
| 余热回流 | [[DATA_天赋池#余热回流]] | 超频普攻命中后返还能量，验证回流关键词 |

这三个天赋都保持“候选”状态。P0-6 的实现用于验证机制可行性，不代表它们已经进入正式天赋池。

### 3.1 裂纹斩

玩家可见：

```text
裂纹斩
[一级] [普攻] [裂纹]
普攻命中敌人时施加裂纹。裂纹达到 3 层时爆发，造成额外伤害。
```

规则：

- 普攻命中敌人时，对该敌人施加 1 层裂纹。
- 同一个敌人身上的裂纹达到 3 层时，触发裂纹爆发。
- 裂纹爆发造成 10 点额外伤害。
- 裂纹爆发后清除该敌人身上的裂纹层数。

P0-6 使用 3 层测试值。[[DATA_天赋池#裂纹斩]] 中的候选设想是 5 层，但锈犬约 4 刀击杀，5 层难以在 P0 测试环境中触发。P0-6 先用 3 层验证机制，后续再回到 DATA 文档调整正式数值。

设计目的：

裂纹斩用于验证“敌人身上可叠加状态”的天赋效果。它让普攻从单纯扣血变成“持续命中同一目标 → 触发额外收益”的小循环。

### 3.2 护势格挡

玩家可见：

```text
护势格挡
[一级] [弹反] [护势]
完美弹反后获得护势。护势会减少下一次受到的伤害。
```

规则：

- 完美弹反成功后，玩家获得 1 层护势。
- P0-6 中护势最多保留 1 层。
- 玩家受到伤害时，如果拥有护势，则本次伤害减少 3 点，并消耗护势。
- 普通弹反不触发护势。

P0-6 使用固定减伤 3 点，便于在整数伤害中测试。[[DATA_天赋池#护势格挡]] 中的候选表达为减少 30%，后续根据正式伤害体系再统一。

设计目的：

护势格挡用于验证“玩家自身临时状态”的天赋效果。它把完美弹反转化为后续容错，让弹反流获得防守收益。

### 3.3 余热回流

玩家可见：

```text
余热回流
[一级] [超频] [回流]
超频普攻命中敌人时，返还能量。
```

规则：

- 超频普攻命中敌人并完成扣能后，返还 1 点能量。
- 空挥不返还。
- 非超频普攻不返还。

设计目的：

余热回流用于验证“超频消耗后的能量返还”。它让玩家初步感受到超频路线的资源循环。

当前 1 点返还偏保守。该数值作为可测试起点，后续可以比较 1 点、2 点、50% 消耗返还三档手感。

## 4. 玩家体验流程

### 4.1 触发选择

触发条件：

- 玩家拥有至少 1 颗完整光核。
- 玩家与祭坛互动。
- 祭坛成功消耗 1 颗光核。

流程：

1. 祭坛消耗光核。
2. 发出 `talent_selection_requested`。
3. TalentSelectionUI 打开。
4. 暂停或锁定玩家战斗输入。
5. UI 显示 3 个候选天赋。

### 4.2 选择天赋

玩家在三张天赋卡中选择一张。

每张卡至少显示：

- 天赋名。
- 层级。
- 动作归属。
- 关键词。
- 效果文本。
- Tooltip 文本。

确认后：

1. UI 关闭。
2. TalentSystem 应用天赋。
3. RunData 记录该天赋。
4. 发出天赋已选择/已应用事件。
5. 玩家恢复战斗输入。

### 4.3 选择后的反馈

P0-6 只需要占位反馈：

- UI 关闭。
- 控制台或 HUD 占位提示显示已获得天赋名。
- 天赋效果在下一次相关动作中生效。

正式音效、按钮动画、卡牌翻开效果和高级 UI 反馈归入 P0-7 或后续 UI 阶段。

## 5. 具体规则

### 5.1 候选生成规则

P0-6 固定生成三张天赋：

```text
裂纹斩
护势格挡
余热回流
```

不做随机池、不做权重、不做保底、不做排除规则。

固定三选一的原因：

- 方便验证 UI 和效果接入。
- 方便手动测试三个方向。
- 避免在天赋池还未稳定前引入随机问题。

### 5.2 选择规则

- 每次祭坛触发只允许选择 1 个天赋。
- 选择后该天赋写入 `RunData.acquired_talents`。
- P0-6 允许重复打开 UI 时再次看到同一组天赋。
- 同一个天赋是否允许重复获得，P0-6 暂不展开；测试场景中可先避免重复兑换。

### 5.3 生效规则

天赋生效分两类：

| 类型 | P0-6 示例 | 实现方向 |
|------|-----------|----------|
| 事件监听型 | 余热回流、护势格挡 | TalentSystem 监听 EventBus 事件并处理 |
| 目标状态型 | 裂纹斩 | TalentSystem 需要知道普攻击中的敌人 |

裂纹斩需要“命中目标”信息。当前 `attack_landed(damage)` 只提供伤害，不提供敌人节点。因此 P0-6 需要新增一个带目标的命中事件，或在 Player 命中时调用 TalentSystem 的目标接口。

推荐方式：

```text
保留 attack_landed(damage)
新增 attack_hit_target(target, damage)
```

这样不会破坏当前 OverclockSystem 和旧测试，同时能让裂纹斩知道要给哪个敌人叠层。

### 5.4 重开规则

`RunData.acquired_talents` 记录本局天赋拥有权；`TalentSystem` 记录天赋产生的运行时状态。

P0-6 中，`RunData.reset_run()` 清空 `acquired_talents` 并发出 `run_reset`。`TalentSystem` 监听 `run_reset`，清空当前三选一候选和运行时天赋状态：

- 当前候选清空。
- 裂纹层数清空。
- 护势层数清空。
- 回流状态不保留。

天赋拥有权和运行时状态需要同步清理。若只清空已获得天赋，旧天赋留下的 Buff、叠层或候选数据可能残留到下一局。

## 6. 数值与依据

| 项目 | P0-6 数值 | 依据 |
|------|-----------|------|
| 三选一数量 | 3 | 与长期设计一致，选择压力清晰 |
| 裂纹触发层数 | 3 | 锈犬约 4 刀击杀，3 层能在测试中触发 |
| 裂纹爆发伤害 | 10 | 等于一次基础普攻，便于感知额外收益 |
| 护势上限 | 1 层 | 避免早期叠层复杂度 |
| 护势减伤 | 3 点 | 锈犬 8 伤害时可明显感知，但不完全免伤 |
| 余热回流 | 1 能量 | 保守起点，验证回流链路 |

待调数值：

- 余热回流可以测试 1 点 / 2 点 / 本次消耗 50%。
- 护势可测试固定减伤 / 百分比减伤。
- 裂纹触发层数需要根据敌人血量和普攻节奏调整。

## 7. 与其他系统的联系

**与光核与祭坛**：祭坛负责消耗光核并发出天赋选择请求。天赋系统不直接判断祭坛是否可用，也不直接扣光核。→ [[策划案_光核与祭坛]]

**与玩家行为系统**：天赋效果依赖普攻命中、完美弹反、受伤等动作事件。玩家行为系统需要提供清晰事件，避免天赋直接侵入玩家状态机。→ [[策划案_玩家行为系统]]

**与能量系统**：余热回流会在超频普攻完成扣能后返还能量。返还应通过 RunData 能量接口处理，保持 HUD 和信号同步。→ [[策划案_能量系统]]

**与敌人系统**：裂纹斩需要在敌人身上记录裂纹层数，并在层数满时对该敌人造成额外伤害。敌人只需要能承受额外伤害，不需要理解天赋来源。→ [[策划案_锈犬]]

**与天赋池数据**：P0-6 使用 [[DATA_天赋池]] 中的候选天赋，但不会把候选状态改为已确认。实现后再根据测试结果调整状态。

## 8. UI 与文本需求

### 8.1 TalentSelectionUI 最低结构

TalentSelectionUI 需要：

- 半透明背景或占位面板。
- 3 个天赋卡槽。
- 每个卡槽显示天赋名、层级、动作归属、关键词、效果文本。
- 关键词统一使用金色高亮，提示玩家这是可解释的机制词。
- 鼠标悬停整张天赋卡时，在天赋卡旁边显示该卡包含的所有关键词 Tooltip，形式参考《杀戮尖塔》的关键词说明浮窗。
- 鼠标点击或键盘确认选择。
- 选择后关闭 UI。

P0-6 使用简单 Panel + Label + Button 作为占位卡牌表现。Tooltip 范围为基础悬停显示；动画、复杂定位和手柄焦点适配归后续 UI 阶段。

## 9. 程序需求

### 9.1 TalentData Resource

新增 `TalentData` Resource。

字段建议：

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | StringName | 稳定 ID |
| `name_key` | StringName | 多语言名称 key |
| `description_key` | StringName | 多语言描述 key |
| `tier` | int | 1 / 2 / 3 |
| `primary_archetype` | StringName | 主流派 |
| `secondary_archetype` | StringName | 副流派，可为空 |
| `keywords` | Array[StringName] | 关键词列表 |
| `effect_type` | StringName | P0-6 枚举式效果类型 |
| `effect_value` | float | 主要数值 |
| `effect_extra_value` | float | 备用数值 |
| `icon` | Texture2D | 占位图标，可为空 |

P0-6 需要 3 个 `.tres`：

- `crack_slash.tres`
- `guard_momentum.tres`
- `residual_backflow.tres`

### 9.2 P0-6 effect_type

P0-6 只支持 3 种效果：

| effect_type | 对应天赋 | 说明 |
|-------------|----------|------|
| `crack_on_attack` | 裂纹斩 | 普攻命中目标时叠裂纹，满层爆发 |
| `guard_on_perfect_parry` | 护势格挡 | 完美弹反后获得护势 |
| `backflow_on_overclock_attack_hit` | 余热回流 | 超频普攻命中并扣能后返还能量 |

暂不做脚本化任意效果系统。原因是 P0-6 只需要验证闭环，枚举式效果更容易测试和维护。

### 9.3 TalentSystem

新增 TalentSystem，建议挂在测试房间或作为系统节点接入 bootstrap。

职责：

- 监听 `talent_selection_requested`。
- 提供 P0-6 固定三选一候选。
- 接收 UI 选择结果。
- 调用 `RunData.add_talent(talent_data)`。
- 根据已获得天赋监听相关事件并应用效果。
- 管理运行时状态：当前候选、裂纹层数、护势层数，以及后续 Buff / 叠层 / 临时标记。
- 监听 `run_reset`，清空天赋运行时状态。

TalentSystem 不负责：

- 扣光核。
- 打开祭坛。
- 玩家输入状态机。
- 具体 UI 排版。

### 9.4 EventBus 信号

现有信号：

```gdscript
talent_selection_requested
talent_selected(talent_data: Resource)
```

建议新增：

| 信号 | 参数 | 用途 |
|------|------|------|
| `talent_options_generated` | options: Array[Resource] | UI 接收三选一候选 |
| `talent_applied` | talent_data: Resource | 天赋效果已应用 |
| `talent_selection_closed` | — | UI 关闭，恢复输入 |
| `attack_hit_target` | target: Node, damage: int | 裂纹斩需要知道命中目标 |

保留 `attack_landed(damage)`，避免影响现有能量和超频测试。

### 9.5 RunData

RunData 已有：

```gdscript
var acquired_talents: Array[Resource] = []
func add_talent(talent_data: Resource) -> void
```

P0-6 需要确认：

- `reset_run()` 会清空 `acquired_talents`。
- `reset_run()` 会发出 `run_reset`，供 `TalentSystem` 清空运行时状态。
- `add_talent()` 会发出 `talent_selected`。
- 同一局内已选天赋可以被 TalentSystem 查询。

RunData 只保存天赋拥有权。天赋造成的临时 Buff、叠层和战斗状态由 TalentSystem 管理，并通过 `run_reset` 与新一局重置同步。

### 9.6 TalentSelectionUI Tooltip 实现方向

P0-6 的 Tooltip 由整张天赋卡触发。

建议 UI 结构：

```text
TalentSelectionUI
- CardRow
  - TalentCard_1
  - TalentCard_2
  - TalentCard_3
- TooltipStack
  - TooltipPanel_1
  - TooltipPanel_2
```

实现要求：

- 每张 `TalentCard` 根据 `TalentData.keywords` 渲染关键词行。
- 关键词统一使用金色字体。
- `TalentCard` 接收 `mouse_entered` 时，调用 `TalentSelectionUI.show_tooltips_for_card(talent_data, card)`。
- `TalentCard` 接收 `mouse_exited` 时，调用 `TalentSelectionUI.hide_tooltips()`。
- `show_tooltips_for_card()` 读取该卡的所有关键词，按卡面关键词顺序生成 Tooltip。
- Tooltip 显示在当前卡牌旁边。右侧空间不足时，可显示在左侧。
- Tooltip 文本来自关键词多语言 key。
- P0-6 同一时间只显示当前悬停卡牌的 Tooltip。

### 9.7 玩家输入锁定

三选一 UI 打开时，应锁定玩家战斗输入。

P0-6 可用简单方式处理：

- UI 打开时暂停游戏树，或
- 设置一个全局/房间级选择状态，让玩家输入状态机不响应战斗输入。

具体采用哪种方式开发时根据当前场景结构验证。P0-6 的要求是：选择期间玩家不会继续移动、攻击、弹反或受伤导致 UI 测试混乱。

## 10. 验收标准

- [ ] 有 1 颗光核时，祭坛互动会打开三选一天赋 UI。
- [ ] 没有光核时，祭坛不会打开天赋 UI，并保留失败提示。
- [ ] UI 同时显示 `裂纹斩`、`护势格挡`、`余热回流` 三个候选。
- [ ] 每张天赋卡显示名称、层级、动作归属、关键词和效果描述。
- [ ] 关键词统一使用金色高亮。
- [ ] 鼠标悬停整张天赋卡时，天赋卡旁边显示该卡包含的所有关键词 Tooltip。
- [ ] 选择任一天赋后，UI 关闭，并调用 `RunData.add_talent()`。
- [ ] 重开或 `RunData.reset_run()` 后，已获得天赋清空。
- [ ] 重开或 `RunData.reset_run()` 后，天赋运行时状态清空，包括当前候选、裂纹层数和护势层数。
- [ ] 选择裂纹斩后，普攻命中同一敌人 3 次会触发裂纹爆发并造成额外伤害。
- [ ] 选择护势格挡后，完美弹反成功会获得护势，下一次受伤减少伤害并消耗护势。
- [ ] 选择余热回流后，超频普攻命中并扣能后会返还 1 点能量。
- [ ] 未选择对应天赋时，上述三个效果不会触发。
- [ ] 现有 P0-3/P0-4/P0-5 测试不因新增天赋系统而失效。
