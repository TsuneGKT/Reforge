## 1. 目标

本策划案定义 P0 原型表现系统的研发执行方案。目标是用 3/4 俯视块状剪影、武器形体、攻击弧光、弹反格挡弧和基础反馈，让玩家与锈犬的状态机在画面上可读。

P0 原型表现服务三个目标：

1. 玩家能看出自己正在移动、攻击、弹反、闪避、受伤、超频或死亡。
2. 玩家能看出锈犬正在追击、蓄力、扑击、被弹反、受伤或死亡。
3. 攻击范围、弹反行为和关键反馈能直接辅助手感判断。

---

## 2. 当前方案

P0 当前采用 **块状剪影动作原型**。

块状剪影动作原型使用 `Polygon2D`、`ColorRect`、`Sprite2D` 或等价 2D 节点表达角色体积、朝向、武器、攻击弧光、弹反弧、预警和反馈。它服务《哈迪斯》式 3/4 俯视动作读感：角色有体积，敌人有轮廓，动作靠大姿态、弧光、预警和反馈读懂。

P0-7 已经实现过 `Line2D` 火柴人底座。火柴人底座保留为调试表现和历史验证结果。P0-8 的主表现切换为块状剪影，原因：

- 3/4 俯视动作更依赖体积和轮廓。
- 32px 线条火柴人在战斗中姿态变化不够明显。
- 锈犬这类四足敌人用线条表现容易产生杂乱轮廓。
- 块状剪影更接近后续像素 Sprite 的替换方式。

## 3. 基准尺寸

| 项目 | P0-8 基准 |
|------|-----------|
| 逻辑 tile | 16 × 16 px |
| 玩家视觉高度 | 48-64 px，约 3-4 tile |
| 玩家碰撞脚底 | 继续围绕脚底 / 身体中心，不按完整视觉高度碰撞 |
| 锈犬视觉长度 | 48-64 px，约 3-4 tile |
| 锈犬视觉高度 | 24-32 px，约 1.5-2 tile |
| 武器视觉长度 | 32-44 px |
| 攻击弧光半径 | 40-56 px |
| 弹反弧半径 | 36-48 px |

角色视觉高度扩大后，判定尺寸不随视觉等比例放大。P0-8 重点是让动作读感接近 3/4 俯视动作游戏，逻辑碰撞仍以现有玩家、敌人和 Hitbox 参数为准。

## 4. 玩家块状剪影结构

建议玩家原型表现挂在 Player 场景下，作为独立表现子节点。

```text
Player
- PrototypeVisualController
  - ShadowShape
  - BodyRoot
    - BodyShape
    - ChestShape
    - HeadShape
    - FacingMarker
  - WeaponRoot
    - WeaponShape
    - AttackArc
    - ParryArc
  - EffectRoot
    - AfterimageLayer
    - FlashOverlay
    - OverclockPulse
```

节点职责：

| 节点 | 职责 |
|------|------|
| `ShadowShape` | 表达脚底位置和 3/4 俯视落点 |
| `BodyShape` | 表达玩家主要体积 |
| `ChestShape` | 表达上半身朝向 |
| `HeadShape` | 表达头部和朝向 |
| `FacingMarker` | 表达当前面向，可用小色块或头部偏移 |
| `WeaponShape` | 表达武器位置、后摆、挥击和横挡 |
| `AttackArc` | 表达普攻生效范围 |
| `ParryArc` | 表达弹反格挡面 |
| `AfterimageLayer` | 表达闪避残影 |
| `FlashOverlay` | 表达命中、受伤、弹反成功闪光 |
| `OverclockPulse` | 表达超频动作脉冲 |

## 5. 玩家状态姿态表

| 状态 / 事件 | 表现要求 |
|-------------|----------|
| Idle | 身体保持 3/4 俯视体积，头部和胸部朝最后输入方向 |
| Move | 身体略微前倾，脚底阴影跟随移动，保留朝向 |
| Attack 起手 | 武器块后摆，胸部朝攻击方向锁定 |
| Attack 生效 | 武器块前挥，显示大弧光，弧光方向对齐 Hitbox |
| Attack 收束 | 弧光淡出，武器回到身体侧前方 |
| Parry | 武器块横在身前，显示蓝白 / 金色格挡弧 |
| Parry 成功 | 触发短闪、HitStop、弹反音和轻屏震 |
| Dodge | 身体向位移方向压低，生成短残影 |
| Hurt | 身体后仰，整体闪红并产生轻微击退 |
| Dead | 身体块、头部和武器分离散开并淡出 |
| Overclock 动作 | 当前动作叠加金色脉冲，攻击弧光或格挡弧变亮 |

死亡使用统一散开规则：身体块、头部、武器和少量碎片沿身体中心向外偏移，同时降低透明度。死亡表现结束后，角色进入不可输入或不可交互状态。

## 6. 锈犬块状剪影结构

锈犬原型表现挂在 RustHound 场景下，独立于状态机和伤害组件。

```text
RustHound
- PrototypeVisualController
  - ShadowShape
  - BodyRoot
    - BodyShape
    - HeadShape
    - ArmorShape
    - LegHintShapes
  - AttackRoot
    - AttackWarning
    - LungeTrail
  - EffectRoot
    - FlashOverlay
    - BreakApartPieces
```

节点职责：

| 节点 | 职责 |
|------|------|
| `ShadowShape` | 表达锈犬脚底位置和落点 |
| `BodyShape` | 表达低矮横向身体 |
| `HeadShape` | 表达头部朝向和扑击目标方向 |
| `ArmorShape` | 表达犬甲与身体融合 |
| `LegHintShapes` | 用少量短块表达四足身份 |
| `AttackWarning` | 表达扑击方向和前摇预警 |
| `LungeTrail` | 表达扑击速度和方向 |
| `FlashOverlay` | 表达受伤和被弹反反馈 |
| `BreakApartPieces` | 表达死亡散开 |

---

## 7. 锈犬状态姿态表

| 状态 / 事件 | 表现要求 |
|-------------|----------|
| Idle / Patrol | 低矮横向身体，头部朝巡逻方向 |
| Chase | 身体前倾，头部和背甲朝玩家方向 |
| Charge | 身体后压，头部压低，显示扑击方向预警 |
| Lunge | 身体拉长，沿锁定方向扑出，显示短拖影 |
| ParriedStun | 身体被弹开后压低或侧翻，触发蓝白 / 金色闪光 |
| Hurt | 整体闪白，保留当前行为节奏 |
| Dead | 身体块、头部和装甲片分离散开并淡出，关闭移动和攻击判定 |

锈犬 Charge 与 Lunge 是优先级最高的表现。Charge 负责让玩家读到前摇，Lunge 负责让玩家判断弹反或闪避时机。

## 8. 事件对接

原型表现系统监听状态机事件或 EventBus 信号，不直接决定战斗规则。

玩家侧建议对接：

| 来源 | 表现 |
|------|------|
| 进入 Idle / Move | 切换基础姿态 |
| `attack_started` | 播放攻击起手姿态 |
| 攻击生效阶段 | 显示攻击弧光 |
| 攻击结束 | 隐藏攻击弧光，回到收势 |
| `parry_started` | 显示格挡弧 |
| `parry_succeeded(is_perfect)` | 播放弹反成功反馈 |
| `dodge_started` | 播放闪避压低和残影 |
| 玩家受伤事件 | 播放闪红和后仰 |
| `overclock_action_triggered(action_type, cost)` | 播放金色脉冲 |
| 玩家死亡 | 播放块状散开 |

锈犬侧建议对接：

| 来源 | 表现 |
|------|------|
| 进入 Chase | 播放前倾追击姿态 |
| 进入 Charge | 播放后蹲蓄力和攻击预警 |
| 进入 Lunge | 播放身体拉伸和拖影 |
| 进入 ParriedStun | 播放被弹反弹开和压低 / 侧翻 |
| `take_damage()` | 播放闪白 |
| 死亡 | 播放块状散开 |

## 9. P0-8 实现细化

P0-7 已完成原型表现底座：`PrototypeVisualController`、`FeedbackSystem`、`HitStopController` 和 `ScreenShakeController` 已经存在。P0-8 当前重点是用块状剪影替换火柴人主表现。

### 9.1 当前基础

`PrototypeVisualController` 已具备以下能力：

| 能力 | 当前表现 |
|------|----------|
| Idle | Line2D 火柴人姿态 |
| Move | Line2D 腿部线条交替 |
| Attack 起手 | 武器线后摆 |
| Attack 生效 | 显示攻击弧光 |
| Parry | 显示弹反格挡弧 |
| Hurt | 红色闪光 |
| Overclock | 金色脉冲 |

P0-8 在此基础上调整：

- 玩家从 Line2D 火柴人切换为块状剪影。
- 玩家视觉高度提升到 48-64 px。
- 攻击弧光随角色尺寸扩大，并继续对齐 Hitbox。
- 弹反格挡弧随角色尺寸扩大。
- 火柴人节点可保留为调试开关。
- 锈犬采用低矮块状剪影。

### 9.2 玩家块状剪影实验

先只改玩家，验证方向是否成立：

1. 在 Player 场景中添加块状剪影表现节点。
2. 保留现有攻击、弹反、移动、受伤和死亡规则。
3. Idle / Move 显示身体体积、头部朝向和脚底阴影。
4. Attack 起手显示武器后摆。
5. Attack 生效显示大弧光。
6. Parry 显示身前格挡弧。
7. Hurt 显示红闪和后仰。
8. Overclock 显示金色脉冲。
9. Dead 显示块状散开。

玩家块状剪影实验通过后，再推进锈犬块状剪影。

### 9.3 锈犬块状剪影接入

1. 给 RustHound 场景接入独立的块状剪影表现节点。
2. 保留现有伤害、追击、扑击、弹反和死亡规则。
3. Chase 进入时切换前倾追击姿态。
4. Charge 进入时切换后蹲蓄力姿态，并显示扑击方向预警。
5. Lunge 进入时拉长身体，显示扑击拖影。
6. ParriedStun 进入时播放弹开、压低 / 侧翻和蓝白 / 金色闪光。
7. Hurt 事件播放闪白。
8. Dead 播放块状散开并关闭判定。

### 9.4 P0-8 测试重点

- 玩家块状剪影是否比火柴人更接近 3/4 俯视动作读感。
- 不看调试文字时，玩家 Idle / Move / Attack / Parry / Dodge / Hurt / Dead / Overclock 是否能区分。
- 攻击弧光是否朝向实际攻击方向。
- 攻击弧光是否基本覆盖实际 Hitbox。
- 弹反格挡弧是否表达玩家正在防御。
- 锈犬 Charge 是否能提前读到扑击方向。
- 锈犬 Lunge 是否能读到扑击路径。
- 锈犬被弹反和死亡是否有明确结果反馈。
- 原型表现不会改变伤害、弹反、移动、死亡和天赋效果规则。

## 10. 验收标准

- [ ] 玩家块状剪影取代火柴人作为主要角色表现。
- [ ] 玩家在 Idle / Move / Attack / Parry / Dodge / Hurt / Dead / Overclock 中有可区分姿态。
- [ ] 普攻生效阶段显示武器弧光，方向与实际攻击判定一致。
- [ ] 弹反阶段显示格挡弧，弹反成功有短闪。
- [ ] 玩家死亡时身体块、头部和武器散开并淡出。
- [ ] 锈犬 Chase / Charge / Lunge / ParriedStun / Hurt / Dead 有可区分姿态。
- [ ] 锈犬 Charge 能让玩家提前读到扑击方向。
- [ ] 锈犬 Lunge 能让玩家判断扑击路径。
- [ ] 锈犬死亡时身体块、头部和装甲片散开并关闭判定。
- [ ] 原型表现系统只消费状态和事件，不改变伤害、移动、弹反和死亡规则。
