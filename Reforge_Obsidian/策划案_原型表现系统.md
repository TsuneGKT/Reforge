## 1. 目标

本策划案定义 P0 原型表现系统的研发执行方案。目标是用 Line2D 火柴人、武器线、攻击弧光、弹反格挡弧和基础反馈，让玩家与锈犬的状态机在画面上可读。

P0 原型表现服务三个目标：

1. 玩家能看出自己正在移动、攻击、弹反、闪避、受伤、超频或死亡。
2. 玩家能看出锈犬正在追击、蓄力、扑击、被弹反、受伤或死亡。
3. 攻击范围、弹反行为和关键反馈能直接辅助手感判断。

---

## 2. 实现范围

P0 采用方案 A：固定 Line2D 火柴人。

方案 A 使用固定节点和预设姿态表达状态。脚本根据状态事件切换各条线的端点、颜色、显示隐藏和透明度。

方案 B：关节点骨架火柴人保留为后续升级方向。方案 B 可以把肩、肘、手、髋、膝、脚作为独立关节点，再由 Line2D 连接，适合后续做更细的姿态插值和动画复用。

P0 先实现方案 A，原因：

- 节点结构直接。
- 姿态清楚。
- 方便和现有状态机对接。
- 后续替换正式 Sprite 或升级骨架时，表现层仍可复用状态事件。

## 3. 玩家火柴人节点结构

建议玩家原型表现挂在 Player 场景下，作为独立表现子节点。

```text
Player
- PrototypeVisualController
  - BodyRoot
    - HeadShape
    - TorsoLine
    - LeftArmLine
    - RightArmLine
    - LeftLegLine
    - RightLegLine
  - WeaponPivot
    - WeaponLine
    - AttackArc
    - ParryArc
  - EffectRoot
    - AfterimageLayer
    - FlashOverlay
```

节点职责：

| 节点 | 职责 |
|------|------|
| `HeadShape` | 表达头部和朝向 |
| `TorsoLine` | 表达身体方向 |
| `LeftArmLine` / `RightArmLine` | 表达攻击、弹反、受伤姿态 |
| `LeftLegLine` / `RightLegLine` | 表达移动、闪避、受伤姿态 |
| `WeaponLine` | 表达武器方向和收势 |
| `AttackArc` | 表达普攻生效范围 |
| `ParryArc` | 表达弹反格挡面 |
| `AfterimageLayer` | 表达闪避和超频残影 |
| `FlashOverlay` | 表达命中、受伤、弹反成功闪光 |

## 4. 玩家状态姿态表

| 状态 / 事件 | 表现要求 |
|-------------|----------|
| Idle | 身体朝最后输入方向，武器收在身体侧前方 |
| Move | 身体保持朝向，腿部线条轻微交替，身体可轻微上下浮动 |
| Attack 起手 | 武器线后摆，攻击方向锁定 |
| Attack 生效 | 武器线前挥，显示攻击弧光，弧光方向对齐 Hitbox |
| Attack 收束 | 攻击弧光淡出，武器线回到收势位置 |
| Parry | 武器线横在身体前方，显示弹反格挡弧 |
| Parry 成功 | 触发短闪、HitStop、弹反音和轻屏震 |
| Dodge | 身体向位移方向倾斜，生成短残影 |
| Hurt | 身体后仰，整体闪红并产生轻微击退 |
| Dead | 线条向外散开并淡出，输入停止 |
| Overclock 动作 | 当前动作叠加金色脉冲，攻击弧光或格挡弧变亮 |

死亡线条散开使用统一规则：各条 Line2D 的端点沿身体中心向外偏移，同时降低透明度。死亡表现结束后，角色进入不可输入或不可交互状态。

## 5. 锈犬火柴原型节点结构

锈犬原型表现挂在 RustHound 场景下，独立于状态机和伤害组件。

```text
RustHound
- PrototypeVisualController
  - BodyRoot
    - BodyLine
    - HeadShape
    - FrontLegLine
    - BackLegLine
    - ArmorPlateShape
  - AttackRoot
    - AttackWarning
    - LungeTrail
  - EffectRoot
    - FlashOverlay
```

节点职责：

| 节点 | 职责 |
|------|------|
| `BodyLine` | 表达锈犬身体方向和拉伸 |
| `HeadShape` | 表达头部朝向和扑击目标方向 |
| `FrontLegLine` / `BackLegLine` | 表达追击、后蹲、扑击和趴伏 |
| `ArmorPlateShape` | 表达锈犬铁甲身份 |
| `AttackWarning` | 表达扑击方向和前摇预警 |
| `LungeTrail` | 表达扑击速度和方向 |
| `FlashOverlay` | 表达受伤、被弹反和死亡反馈 |

## 6. 锈犬状态姿态表

| 状态 / 事件 | 表现要求 |
|-------------|----------|
| Idle / Patrol | 身体低伏，保留军犬轮廓 |
| Chase | 身体前倾，腿部线条拉开，朝玩家移动 |
| Charge | 身体后压，头部压低，显示扑击方向预警 |
| Lunge | 身体线拉长，沿锁定方向扑出，显示短拖影 |
| ParriedStun | 身体被弹开后压低或趴伏，触发蓝白 / 金色闪光 |
| Hurt | 整体闪白，保持当前行为节奏 |
| Dead | 线条向外散开并淡出，关闭移动和攻击判定 |

锈犬 Charge 与 Lunge 是优先级最高的表现。Charge 负责让玩家读到前摇，Lunge 负责让玩家判断弹反或闪避时机。

## 7. 事件对接

原型表现系统监听状态机事件或 EventBus 信号，不直接决定战斗规则。

玩家侧建议对接：

| 来源 | 表现 |
|------|------|
| 进入 Idle / Move | 切换基础姿态 |
| `attack_started` | 播放攻击起手姿态 |
| 攻击生效阶段 | 显示攻击弧光 |
| 攻击结束 | 隐藏攻击弧光，回到收势 |
| `parry_started` | 显示格挡弧 |
| `parry_succeeded(is_perfect)` | 播放弹反成功反馈 |
| `dodge_started` | 播放闪避倾斜和残影 |
| 玩家受伤事件 | 播放闪红和后仰 |
| `overclock_action_triggered(action_type, cost)` | 播放金色脉冲 |
| 玩家死亡 | 播放线条散开 |

锈犬侧建议对接：

| 来源 | 表现 |
|------|------|
| 进入 Chase | 播放前倾追击姿态 |
| 进入 Charge | 播放后蹲蓄力和攻击预警 |
| 进入 Lunge | 播放身体拉伸和拖影 |
| 进入 ParriedStun | 播放被弹反弹开和趴伏 |
| `take_damage()` | 播放闪白 |
| 死亡 | 播放线条散开 |

## 8. 实现顺序

1. 建立 `PrototypeVisualController` 基类或通用脚本。
2. 在 Player 场景中接入玩家 Line2D 火柴人节点。
3. 接入 Idle / Move / Attack / Parry 四个玩家基础姿态。
4. 接入攻击弧光和弹反格挡弧。
5. 接入 Dodge / Hurt / Overclock / Dead 表现。
6. 在 RustHound 场景中接入锈犬 Line2D 火柴原型。
7. 接入 Chase / Charge / Lunge / ParriedStun / Hurt / Dead 表现。
8. 增加状态机可视化测试或调试场景。

## 9. P0-8 实现细化

P0-7 已完成原型表现底座：`PrototypeVisualController`、`FeedbackSystem`、`HitStopController` 和 `ScreenShakeController` 已经存在。P0-8 的重点是把表现底座接入玩家和锈犬，让状态机变化能在画面中被看见。

### 9.1 当前基础

`PrototypeVisualController` 已具备以下能力：

| 能力 | 当前表现 |
|------|----------|
| Idle | 默认火柴人姿态 |
| Move | 腿部线条交替 |
| Attack 起手 | 武器线后摆 |
| Attack 生效 | 显示攻击弧光 |
| Parry | 显示弹反格挡弧 |
| Hurt | 红色闪光 |
| Overclock | 金色脉冲 |

P0-8 在此基础上补齐：

- 玩家场景接入 `PrototypeVisualController`。
- 玩家状态机调用对应表现接口。
- 攻击弧光和玩家攻击方向、攻击判定对齐。
- 弹反格挡弧和玩家弹反方向对齐。
- 闪避、死亡表现补齐。
- 锈犬接入 Line2D 火柴犬表现。
- 锈犬 Charge / Lunge / ParriedStun / Hurt / Dead 与状态机同步。

### 9.2 P0-8 视觉参数

| 项目 | P0-8 参数 |
|------|-----------|
| 火柴人线宽 | 3 px |
| 玩家头部高度 | 头部中心约在身体中心上方 17 px |
| 玩家整体视觉高度 | 约 32 px |
| 武器线长度 | 约 20-26 px |
| 攻击弧光半径 | 28 px |
| 攻击弧光角度 | 约 65 度 |
| 弹反格挡弧半径 | 24 px |
| 弹反格挡弧角度 | 约 100 度 |
| 受伤闪红时间 | 约 0.14 秒 |
| 超频金色脉冲时间 | 约 0.16 秒 |
| 死亡散开时间 | 约 0.35-0.5 秒 |

这些参数服务原型可读性。研发时可根据实际 Hitbox 和镜头观感微调，调整后同步回本表。

### 9.3 玩家接入顺序

1. 在 Player 场景中挂载 `PrototypeVisualController`。
2. 隐藏或弱化旧的方向箭头 / 几何占位，让火柴人作为主要角色表现。
3. Idle / Move 调用 `play_idle()` / `play_move(direction)`。
4. Attack 起手调用 `play_attack_start(direction)`。
5. Attack 生效调用 `show_attack_arc(direction)`。
6. Attack 结束调用 `hide_attack_arc()` 并回到收势。
7. Parry 开始调用 `show_parry_arc(direction)`。
8. Parry 结束调用 `hide_parry_arc()`。
9. Parry 成功通过 `FeedbackSystem` 触发 HitStop、屏震和闪光。
10. Dodge 调用闪避倾斜或残影表现。
11. Hurt 调用 `play_hurt_flash()`。
12. Overclock 动作触发调用 `play_overclock_pulse()`。
13. Dead 调用线条散开表现。

### 9.4 锈犬接入顺序

1. 给 RustHound 场景接入独立的火柴犬表现节点或锈犬专用 `PrototypeVisualController`。
2. 保留现有伤害、追击、扑击、弹反和死亡规则。
3. Chase 进入时切换前倾追击姿态。
4. Charge 进入时切换后蹲蓄力姿态，并显示扑击方向预警。
5. Lunge 进入时拉长身体线，显示扑击拖影。
6. ParriedStun 进入时播放弹开、趴伏和蓝白 / 金色闪光。
7. Hurt 事件播放闪白。
8. Dead 播放线条散开并关闭判定。

### 9.5 P0-8 测试重点

- 玩家火柴人是否取代旧箭头成为主要可读对象。
- 不看调试文字时，玩家 Idle / Move / Attack / Parry / Dodge / Hurt / Dead / Overclock 是否能区分。
- 攻击弧光是否朝向实际攻击方向。
- 攻击弧光是否基本覆盖实际 Hitbox。
- 弹反格挡弧是否表达玩家正在防御。
- 锈犬 Charge 是否能提前读到扑击方向。
- 锈犬 Lunge 是否能读到扑击路径。
- 锈犬被弹反和死亡是否有明确结果反馈。
- 原型表现不会改变伤害、弹反、移动、死亡和天赋效果规则。

## 10. 验收标准

- [ ] 玩家在 Idle / Move / Attack / Parry / Dodge / Hurt / Dead / Overclock 中有可区分姿态。
- [ ] 普攻生效阶段显示武器弧光，方向与实际攻击判定一致。
- [ ] 弹反阶段显示格挡弧，弹反成功有短闪。
- [ ] 玩家死亡时线条散开并淡出。
- [ ] 锈犬 Chase / Charge / Lunge / ParriedStun / Hurt / Dead 有可区分姿态。
- [ ] 锈犬 Charge 能让玩家提前读到扑击方向。
- [ ] 锈犬 Lunge 能让玩家判断扑击路径。
- [ ] 锈犬死亡时线条散开并关闭判定。
- [ ] 原型表现系统只消费状态和事件，不改变伤害、移动、弹反和死亡规则。
