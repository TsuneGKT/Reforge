## 一、定位

本文记录 Reforge P0 后半段的美术基准。目标是让一间房的完整流程具备可展示形态：玩家、锈犬、房间、交互物和基础反馈在同一画面里风格统一，玩家能看清动作状态和战斗信息。

P0 美术采用 **3/4 俯视简化手绘动作风**。资产先服务可读性和流程展示，后续再根据实际效果决定是否像素化、精修或替换。

---

## 二、风格原则

| 原则 | 要求 |
|------|------|
| 人物简化 | 不依赖复杂五官、小饰品和碎纹理，靠头身比例、装备大形和武器识别角色 |
| 强轮廓 | 角色、敌人、武器和交互物需要从地面中分离出来 |
| 大色块 | 身体、甲片、武器、阴影使用清楚的大块面，减少细碎材质 |
| 低帧数可读 | 动画依靠关键姿势、前摇、弧光和特效读懂，不依赖高帧数细腻过渡 |
| 3/4 俯视统一 | 能看到头顶、肩部和身体正面一点点，脚底位置作为碰撞与排序基准 |

参考方向是“简化版 3/4 动作游戏角色”：保留《哈迪斯》式的俯视动作读感，降低细节密度和绘制成本。角色不绑定复杂世界观符号，先保证在游戏内能动、能打、能被读懂。

## 三、基准尺寸

| 项目 | P0 美术基准 |
|------|-------------|
| 逻辑 tile | 32 × 32 px |
| 推荐视口 | 1280 × 720 px |
| 标准房间 | 64 × 36 tile，约 2048 × 1152 px |
| 摄像机可视战斗区 | 约 40 × 22.5 tile |
| 玩家显示高度 | 当前用 512 画布资产和 `visual_scale = 0.394` 校准，主体约 141 px 高 |
| 玩家脚底碰撞 | 约 32-40 px 宽、20-28 px 高 |
| 普通敌人视觉尺寸 | 当前锈犬显示基准约 117 px 宽 |
| 精英 / 小 Boss 视觉尺寸 | 约 192-256 px |

标准房间用于 P0 完整关卡展示。这个尺寸能容纳怪物刷新点、结晶点、祭坛、边缘装饰和少量探索空间，同时让摄像机无法一次看完整个房间，形成更接近大房间战斗场的空间感。

## 四、分辨率与全屏适配

Reforge 的核心画面以 16:9 为基准设计，当前推荐视口为 1280 × 720。这个基准服务 PC / Steam 的主流显示环境，例如 1920 × 1080、2560 × 1440 和 3840 × 2160。

不同显示器比例会出现三种处理方式：

| 方案 | 结果 | P0 判断 |
|------|------|---------|
| `keep` | 保持 16:9，屏幕比例不一致时出现黑边 | 稳定但不利于当前 Mac 全屏校验 |
| `expand` | 保持画面不变形，屏幕多出的方向显示更多房间内容 | P0 校准阶段优先采用 |
| 裁切填满 | 去掉黑边，但切掉部分画面 | 不采用 |

P0 美术和地图先按 `expand` 校验全屏观感。这样在非 16:9 屏幕上不会拉伸变形，也能避免黑边；代价是不同屏幕能看到的边缘空间略有差异。房间边缘、刷新点、UI 和装饰需要预留安全区，不把关键战斗信息放在只对某些屏幕可见的位置。

当前 Mac 内屏为 2560 × 1664。1280 × 720 画面在 `keep` 下会以 2560 × 1440 显示，上下留黑边；在 `expand` 下会铺满屏幕，并显示更多竖向房间内容。

## 七、首批资产清单

P0 先制作一间房、一名玩家、一只锈犬和一套基础交互物。

| 资产 | 首批需求 |
|------|----------|
| 玩家 | Idle、Move、Attack、Parry、Dodge、Hurt、Dead 的关键帧或低帧动画 |
| 锈犬 | Idle / Patrol、Chase、Charge、Lunge、ParriedStun、Hurt、Dead |
| 攻击表现 | 玩家挥砍弧光、弹反格挡弧、命中闪光；攻击动画按 [[ART_攻击动画分层规范]] 分层接入 |
| 地图 | 64 × 36 tile 房间，包含地面、墙体、边缘装饰和碰撞边界 |
| 交互物 | 结晶点、祭坛、出口或房间边界标识 |
| UI / 反馈 | 生命、能量、光核、天赋选择的占位风格统一 |

## 八、AI 资产工作流

当前采用以下流程试制资产：

```text
Codex / GPT 生单帧、首尾帧或关键姿势参考图
→ 即梦根据参考图生成动作视频
→ Blender 或剪辑工具导出分帧图
→ PS 清理、缩小、统一边缘和色块
→ Aseprite 合成精灵图
→ Godot 导入 SpriteFrames / AnimationPlayer
```

每次进入动画制作前，先用单帧校准风格、比例和朝向。单帧通过后再做动作，避免在错误风格上消耗时间。

玩家攻击动画先按 [[ART_攻击动画分层规范]] 制作右下方向普攻样例。角色本体、弧光和 Hitbox 在 Godot 中对齐后，再扩展其他方向和动作。

## 九、当前待验证

1. 玩家 512 × 512 测试精灵在 `visual_scale = 0.394` 下，轮廓和武器是否仍然清楚。
2. 锈犬 512 × 512 测试精灵与玩家放在同一画面时，体型比例是否舒服。
3. `VisualScaleCalibration` 场景中的 64 × 36 tile 房间在 1280 × 720 视口下是否既有大房间感，又不会显空。
4. AI 生成动作视频抽帧后，角色体型和武器长度是否能保持一致。
