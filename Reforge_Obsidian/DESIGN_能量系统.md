## 一、系统定位

能量系统是 Reforge 的核心资源系统。它连接玩家操作、战斗生存和天赋构筑：玩家在战斗中通过弹反获得能量，可以当场消耗能量强化动作，也可以在战斗结束后把剩余能量结晶为光核，用于祭坛兑换天赋。

能量不是单纯的“蓝条”。它同时承担两种价值：

| 用途 | 作用 | 性质 |
|------|------|------|
| 超频动作 | 消耗能量，强化具体动作，用于保命或快速清场 | 即时战斗收益 |
| 光核结晶 | 战斗结束后转化为光核进度，用于兑换天赋 | 长期构筑收益 |

因此，能量系统的核心问题不是“玩家能放几次技能”，而是：**这一点能量现在花掉，还是留到战后变成构筑进度？**

---

## 二、设计目标

### 2.1 生存 vs 成长

玩家在战斗中持续面对同一个选择：消耗能量强化动作，降低当前战斗风险；还是尽量少花能量，把剩余能量留到战斗结束后结晶成光核。

这个选择让操作水平直接影响构筑速度。操作越好，越少需要用超频动作救场，战斗结束后剩余能量越多，光核积累越快，天赋成型越早。

### 2.2 超频必须有诱惑力

如果超频性价比不足，玩家会倾向于永远囤能量换天赋，能量系统就只剩“攒进度条”。因此 P0 阶段的超频动作必须做到：**用了以后明显能救命或清场**。

数值上，P0 先采用偏低消耗；体验上，超频动作要有明确反馈，让玩家感到“这一下值得花能量”。

### 2.3 结晶不打断战斗

结晶只发生在战斗结束后。战斗中能量只服务战斗决策，不弹出菜单、不打断节奏、不让玩家在危险中处理成长结算。

战斗结束后，结晶通过空间行为完成：走到结晶点互动就结晶，不互动就保留能量。

## 三、系统分层

能量系统分为五个职责边界：

| 模块 | 层级 | 职责 |
|------|------|------|
| RunData | 资源层 | 保存 `energy`、`max_energy`、`light_core_progress`、`light_cores`，提供加减、消耗、结晶接口 |
| EnergySystem | 基础能量结算层 | 监听基础动作事件，把弹反成功等事件转化为能量收入 |
| OverclockSystem | 超频动作强化层 | 判断当前动作是否请求超频，处理超频动作消耗、强化和能量不足降级 |
| Player | 动作事件来源 | 执行动作并发出“弹反成功 / 攻击命中 / 闪避触发 / 请求超频动作”等事件 |
| HUD | 显示层 | 监听能量、光核变化并刷新 UI，不参与规则判断 |

### 3.1 RunData：资源层

RunData 是单局数据仓库，只负责存取数值，不判断“为什么加能量”或“为什么扣能量”。

核心接口：

| 接口 | 说明 |
|------|------|
| `add_energy(amount)` | 增加能量，夹到 `0 ~ max_energy`，广播 `energy_changed` |
| `consume_energy(amount) -> bool` | 消耗能量，不足返回 `false` |
| `crystallize()` | 将当前能量转化为光核进度，能量清零 |
| `spend_light_core() -> bool` | 花费 1 颗光核，供祭坛兑换天赋 |

### 3.2 EnergySystem：基础能量结算层

EnergySystem 负责所有“非超频也存在”的能量规则。P0 阶段主要是弹反回能：

- 完美弹反：增加 12 能量
- 普通弹反：增加 6 能量

EnergySystem 不保存长期数据，也不更新 HUD。它监听玩家行为事件，然后调用 RunData。

### 3.3 OverclockSystem：超频动作强化层

OverclockSystem 负责超频动作。玩家按住超频键并执行具体动作时，系统尝试触发该动作的超频版本。

基础输入逻辑：

| 输入 | 行为 |
|------|------|
| 普通攻击 | 普通普攻 |
| 超频键 + 普通攻击 | 超频普攻 |
| 闪避 | 普通闪避 |
| 超频键 + 闪避 | 超频闪避 |
| 弹反 | 普通弹反 |
| 超频键 + 弹反 | 超频弹反 |

玩家可以逐次决定某个动作是否值得消耗能量强化。例如玩家本来想保留能量做超频弹反，但临时必须闪避时，只要不按超频键，闪避就是普通闪避，不会消耗超频能量。

## 四、能量获取

### 4.1 P0 基础获取：弹反回能

P0 阶段，弹反是主要能量输入口。

| 行为 | 能量收益 | 设计意图 |
|------|---------:|----------|
| 完美弹反 | +12 | 8-10 次完美弹反可攒满一池，奖励精准操作 |
| 普通弹反 | +6 | 完美收益的一半，降低新手挫败但保留高手优势 |

这组数值让弹反既是操作奖励，也是构筑节奏的起点。玩家越稳定地弹反，越快获得超频预算和光核结晶进度。

### 4.2 后续扩展

中后期天赋可以扩展能量输入方式，例如：

- 普攻命中回能
- 完美闪避回能
- 技能命中回能
- 超频击杀回能
- 弹反收益倍率提升

这些扩展应通过天赋或后续系统接入，不应改变 P0 的基础原则：前期由弹反启动能量循环。

## 五、超频动作

### 5.1 规则定义

超频是动作强化层。玩家按住超频键并执行动作时，系统尝试触发该动作的超频版本。

核心规则：

- 按住超频键本身不消耗能量。
- 只有超频动作成功触发时才扣能量。
- 能量不足时，动作照常执行普通版，不强化、不扣能。
- 普攻空挥不扣能量；超频普攻只有命中后扣能。
- 超频弹反失败不扣能量；只有弹反成功后扣能。
- 超频闪避触发即扣能，因为闪避收益在位移和无敌开始时已经发生。

后续可通过天赋解锁“超频锁定”一类自动强化规则，用于降低超频流高频使用超频动作时的输入负担。该机制属于中后期超频流扩展，具体规则暂不在 P0 展开。

### 5.2 P0 初始消耗

P0 阶段先取低消耗，避免玩家因为性价比焦虑而完全不用超频。

| 超频动作 | 消耗 | 扣能时机 | P0 强化方向 |
|----------|-----:|----------|-------------|
| 超频普攻 | 3 | 命中敌人后 | 伤害提高，清怪更快 |
| 超频闪避 | 8 | 闪避触发时 | 位移或无敌强化，用于保命 |
| 超频弹反 | 15 | 弹反成功后 | 反击伤害、范围或硬直强化 |

如果 playtest 中玩家仍然倾向于不用超频，优先加强超频效果，而不是继续降低消耗。消耗过低会削弱“生存 vs 成长”的张力。

### 5.3 结算顺序

同一个动作同时产生基础收益和超频消耗时，采用：

```text
基础收益先结算 → 超频消耗后结算
```

例如超频弹反：

| 行为 | 基础收益 | 超频消耗 | 净变化 |
|------|---------:|---------:|-------:|
| 超频完美弹反 | +12 | -15 | -3 |
| 超频普通弹反 | +6 | -15 | -9 |

这样可以保留“弹反本身仍是能量输入口”的理解，同时让超频弹反成为“花少量能量买高收益反击”的选择。

## 六、光核结晶

### 6.1 结晶规则

战斗房间清空后，房间内出现结晶点。玩家与结晶点互动时，将当前所有能量转化为光核进度，并清空当前能量。

```text
当前能量 / 最大能量 = 光核进度增量
```

基础比例：

| 项目 | 数值 |
|------|------|
| 能量池上限 | 100 |
| 光核兑换比例 | 100 能量 = 1 颗光核 |
| 光核存储上限 | P0/P1 前期为 1 颗 |
| 天赋兑换成本 | 1 颗完整光核 |

### 6.2 互动与保留

结晶不是自动发生的。

- 玩家互动结晶点：当前能量全部结晶，能量清零。
- 玩家不互动结晶点：能量保留，不发生结晶。
- 光核已满时：结晶点不可互动，或提示“光核已满”，避免玩家误清空能量。

即使 P0 只有一个测试房间，也要保留“不结晶”的分支。玩家不去结晶点，能量就留在身上；这样可以从第一版开始验证“结晶是主动选择，而非自动结算”。

## 七、HUD 显示

HUD 跟随功能增量实现，不提前做完整界面。

| 阶段 | HUD 内容 | 说明 |
|------|----------|------|
| P0-3 能量系统 | 能量条，建议附带 `current / max` 数字 | 显示当前战斗资源 |
| P0-5 光核结晶 + 祭坛 | 光核数量与光核进度 | 显示构筑进度 |
| P0-6 天赋系统 | 天赋选择 UI、已选天赋显示 | 显示构筑结果 |
| P1 天赋扩展 | 下一次超频动作标识 | 当天赋或自动强化规则使“下一次动作视为超频动作”时，在头像、血条或能量条下方显示清晰标识 |

HUD 只监听 `energy_changed`、`light_core_changed` 等信号，不监听玩家动作来直接改数值。这样以后能量来源变多，HUD 仍然只关心“当前是多少”。

“下一次超频动作标识”用于表达一种待消耗的战斗承诺：玩家看到标识后，应能立即理解下一次普攻、闪避或弹反可能触发超频版本。标识在超频动作触发、超时、被取消或因能量不足降级后消失。

## 八、P0 实现边界

### 8.1 P0-3 必须实现

- EnergySystem 监听 `parry_succeeded`，根据完美/普通弹反增加能量。
- OverclockSystem 识别“超频键 + 动作”的超频请求。
- 超频普攻、超频闪避、超频弹反按 P0 初始数值扣能。
- 能量不足时，动作降级为普通版。
- 能量 HUD 显示当前能量。

### 8.2 P0-5 实现

- 清房后出现结晶点。
- 互动结晶点，将当前能量转化为光核进度。
- 不互动则保留能量。
- 光核满时阻止结晶。
- 光核 HUD 显示光核数量和进度。

### 8.3 暂不实现

- 天赋对能量收益/消耗的改写。
- 超频锁定或其他自动强化规则。
- 技能命中回能。
- 多房间保留能量后的复杂路线选择。
- 光核上限提升。
- 商店、事件、复活消耗能量等高级规则。

## 九、待验证问题

1. 超频普攻消耗 3 是否足够便宜，能否鼓励玩家用超频普攻清怪。
2. 超频闪避消耗 8 是否会让玩家愿意用它救命。
3. 超频弹反净亏 3 / 9 是否符合“高回报但有代价”的预期。
4. 玩家是否仍然倾向于囤能量不用超频；如果是，优先加强超频效果而不是继续降消耗。
5. 结晶点“不互动就保留能量”的规则是否足够直观。
6. 光核满时阻止结晶是否需要明确 UI 提示。
7. 能量 HUD 是否足以让玩家理解“战斗资源”和“构筑进度”的关系。
